# 21 深度生成模型

如果说判别模型是"学会判断"，那生成模型就是"学会创造"。本章先建立全景图，后续 22/23/24 三章分别深入自编码器、GAN 与扩散模型。

## 为什么需要生成模型

到第 20 章为止，我们接触的几乎都是**判别模型**（Discriminative Model）——给定输入 $x$，预测标签 $y$。比如分类器判断一张图是不是猫，回归器预测房价。

但世界不只是"分类"和"回归"。我们还想要：

- 给一段文本描述，**生成**对应的图像（Stable Diffusion）
- 给一段旋律，**补全**成完整乐曲（MusicGen）
- 给一段视频，**预测**未来几帧（World Model）
- 给一个分子结构，**设计**出药效更好的变体（AlphaFold、分子生成）
- 从噪声中**创造**从未见过的人脸（StyleGAN）

这些任务都要求模型**理解数据的分布** $p(x)$，而不仅仅是 $p(y|x)$。这就是**生成模型**（Generative Model）的目标。

## 判别模型 vs 生成模型

用概率的语言来区分：

| | 判别模型 | 生成模型 |
|---|---------|---------|
| 目标 | 学习 $p(y\|x)$ | 学习 $p(x)$，或从中采样 |
| 任务 | 分类、回归 | 生成、密度估计、异常检测 |
| 训练数据 | 需要标签 | 往往不需要标签（无监督） |
| 典型模型 | 逻辑回归、CNN、Transformer | VAE、GAN、扩散模型、Flow |

一个直观的比喻：

- **判别模型**是阅卷老师——给学生（输入）打分（标签）
- **生成模型**是作家——能写出符合某种风格（分布）的新文章

判别模型擅长"判断"，生成模型擅长"创造"。两者并不互斥，许多现代模型（如 Diffusion Transformer）既做生成也做判别。

## 显式密度 vs 隐式密度

学习 $p(x)$ 有两大流派：

**显式密度（Explicit Density）**：直接对 $p(x)$ 建模。
- 优点：可以计算任意样本的似然
- 缺点：模型设计受限，往往需要对分布做简化假设

**隐式密度（Implicit Density）**：不显式建模 $p(x)$，只学习"从分布中采样"的机制。
- 优点：模型设计自由
- 缺点：无法直接计算似然

后续章节的 GAN 就属于隐式密度方法（无法计算 $p(x)$），而 VAE、扩散模型属于显式密度。

## 生成模型的五大类

现代深度生成模型可以分为五大类。理解这个全景图比任何单一模型都重要。

### 1. 自回归模型（Autoregressive）

**核心思想**：把 $p(x)$ 分解为条件概率的链式乘积。

$$p(x) = \prod_{i=1}^{n} p(x_i | x_1, x_2, \ldots, x_{i-1})$$

例如，预测一个句子："今天 天气 很好" → 已知"今天"，预测"天气"；已知"今天 天气"，预测"很好"。

**代表工作**：
- PixelRNN / PixelCNN（按像素生成图像）
- WaveNet（音频生成）
- GPT 系列（文本生成，本质就是自回归）

**优点**：训练稳定、似然可精确计算
**缺点**：生成慢（需要逐个采样），长序列容易累积误差

### 2. 归一化流（Normalizing Flow）

**核心思想**：从一个简单分布（如高斯），通过一系列可逆变换 $f$，映射到复杂的数据分布。

$$z \sim p_z(z), \quad x = f(z), \quad p_x(x) = p_z(f^{-1}(x)) \left| \det \frac{\partial f^{-1}}{\partial x} \right|$$

**代表工作**：NICE、RealNVP、Glow、Flow++

**优点**：精确似然、可双向映射
**缺点**：变换必须可逆，限制了架构设计；高维数据上计算量较大

### 3. 变分自编码器（VAE）

**核心思想**：学习数据的潜变量表示 $z$，通过"编码-解码"实现生成。详见第 22 章。

**代表工作**：VAE（Kingma 2013）、β-VAE、VQ-VAE

**优点**：潜空间有意义、训练稳定、可用于异常检测
**缺点**：生成样本往往偏模糊（相比 GAN）

### 4. 生成对抗网络（GAN）

**核心思想**：让两个网络对抗——生成器试图骗过判别器，判别器试图分辨真假。详见第 23 章。

**代表工作**：GAN（Goodfellow 2014）、DCGAN、WGAN、StyleGAN、CycleGAN

**优点**：生成样本清晰锐利、训练效率高
**缺点**：训练不稳定、模式崩溃、无法计算似然

### 5. 扩散模型（Diffusion Models）

**核心思想**：学习"从噪声中逐步去噪"的过程。详见第 24 章。

**代表工作**：DDPM（Ho 2020）、DDIM、Score-based、Stable Diffusion

**优点**：训练稳定、生成质量极佳（当前 SOTA）
**缺点**：采样慢（需要数十到上千步）

## 五大类对比

| 模型 | 密度估计 | 训练稳定性 | 样本质量 | 采样速度 | 典型应用 |
|------|---------|-----------|---------|---------|---------|
| 自回归 | 精确 | 高 | 中 | 慢 | 文本、音频 |
| 归一化流 | 精确 | 中 | 中 | 快 | 密度估计 |
| VAE | 近似 | 高 | 中（略模糊） | 快 | 潜空间、压缩 |
| GAN | 隐式 | 低 | 高 | 快 | 图像、视频 |
| 扩散 | 近似 | 高 | 极高 | 慢 | 图像、视频、音频 |

**当前趋势**：扩散模型 + Transformer 的组合（如 Stable Diffusion 3、Sora）正在成为生成式 AI 的主流。

## 评估指标

生成模型如何评估"生成得好不好"？这是一个开放问题。

### 1. Inception Score（IS）

- 思想：好的生成样本应该"明确属于某类"且"样本多样性高"
- 公式：$IS = \exp\left(\mathbb{E}_x D_{KL}(p(y|x) \| p(y))\right)$
- 优点：计算简单
- 缺点：依赖 Inception 模型（不适合非自然图像）

### 2. FID（Fréchet Inception Distance）

- 思想：用 Inception 网络提取特征，比较真实样本和生成样本的特征分布
- 公式：$FID = \|\mu_r - \mu_g\|^2 + \text{Tr}(\Sigma_r + \Sigma_g - 2(\Sigma_r\Sigma_g)^{1/2})$
- 优点：与人类评价相关性较好
- 缺点：依赖 Inception、样本数敏感

### 3. 其他

- **KID**（Kernel Inception Distance）：FID 的改进版，对样本数不敏感
- **CLIP Score**：用 CLIP 评估文本-图像一致性（用于文生图）
- **Perplexity**：自回归模型专用，越低越好

## 应用全景

生成模型已经渗透到几乎所有领域：

| 领域 | 典型应用 | 代表模型 |
|------|---------|---------|
| 图像生成 | 文本到图像、风格迁移、超分 | Stable Diffusion、DALL-E 3、SRGAN |
| 视频生成 | 短视频、电影特效 | Sora、Runway Gen-3 |
| 音频 | 音乐合成、语音克隆、音频修复 | MusicGen、Bark、AudioLDM |
| 文本 | 写作辅助、代码生成、对话 | GPT-4、Claude、Copilot |
| 分子 | 药物设计、材料发现 | AlphaFold 3、DiffDock |
| 医学 | 病灶合成、跨模态生成 | Med-PaLM 2、HealthGPT |
| 机器人 | World Model、动作生成 | GAIA-1、UniSim |
| 推荐 | 序列生成、协同过滤 | RecGPT |

## 常见误区

1. **认为生成模型就是 GAN**：GAN 只是其中一类，扩散模型和 VAE 同样重要
2. **盲目追求 FID 低**：FID 不是万能的，某些任务（如异常检测）反而需要高 FID
3. **忽略似然估计能力**：很多应用需要概率输出（如医疗诊断），GAN 不适合
4. **混用评估指标**：不同任务用不同指标，不要拿 FID 评价文本
5. **忽视训练成本**：扩散模型虽然好，但采样慢、显存大，工业部署需要权衡

## 小结

| 概念 | 说明 |
|------|------|
| 判别模型 | 学习 $p(y\|x)$，用于分类/回归 |
| 生成模型 | 学习 $p(x)$，用于生成/密度估计 |
| 显式密度 | 直接建模 $p(x)$，可计算似然 |
| 隐式密度 | 学习采样机制，无法计算似然 |
| 五大类 | 自回归、归一化流、VAE、GAN、扩散 |
| 评估指标 | IS、FID、KID、CLIP Score、Perplexity |
