# 25 NLP 基础

从手机里的语音助手，到能帮你写代码、改文章的 ChatGPT，这些几年前还被视为科幻的场景已悄然融入日常生活。这一切的背后，都是**自然语言处理（Natural Language Processing, NLP）**。本章将带你走进 NLP 的世界，了解它是什么、经历了怎样的发展历程，以及为什么大语言模型（LLM）彻底改变了这个领域。

> 本章是 LLM 篇的开篇。如果你已经学过前面的深度学习基础（第 11-24 章），接下来的内容将从**NLP 视角**重新审视这些技术的应用场景。

## 1 什么是 NLP

NLP 是人工智能的一个重要分支，旨在让计算机**理解、解释和生成人类语言**。用通俗的话说，就是教会计算机"读懂"文字、"听懂"语音，并能像人一样"说出"话语。

从技术角度看，NLP 通常包含两大核心能力：

- **自然语言理解（NLU）**——输入语言，输出结构化信息。例如从"帮我订一张明天去上海的机票"中解析出意图"订票"、目的地"上海"、时间"明天"。
- **自然语言生成（NLG）**——输入结构化信息，输出语言。例如天气 App 根据数据生成"北京今日晴，气温 25 度"。

现代大语言模型以统一的"预测下一个 token"为训练目标，在同一模型内同时覆盖理解与生成。

### NLP 的技术层次

为了让计算机由表及里地理解语言，NLP 的处理过程通常划分为四个由浅入深的层次：

```text
层级 4：语用分析 ── 理解语言的真实意图
  │                  "房间里真冷" → 请求关窗
  │
层级 3：语义分析 ── 理解句子的真实含义
  │                  "苹果" → 水果 or 公司？
  │
层级 2：句法分析 ── 分析句子的语法结构
  │                  主谓宾、从句关系
  │
层级 1：词法分析 ── 处理文本的基本单元
                    分词、词性标注
```

每一个层级都建立在前一层级的基础之上。例如，要理解"房间里真冷"的真实意图（语用分析），首先需要正确分词（词法分析），然后理解句法结构，再理解"冷"的语义，最后才能推断出这不是陈述事实而是表达请求。

### 语言的本质挑战

人类语言对于机器来说有三大核心挑战：

**歧义性**：同一个句子可能有多种理解。例如"咬死了猎人的狗"——是狗被咬死了，还是狗咬死了猎人？再如"朝阳"二字，读 `cháo yáng` 是地名，读 `zhāo yáng` 是早晨的太阳。还有结构歧义："两个学校的老师"——是两所学校的老师，还是两个来自同一学校的老师？

**多样性**：同一个意思可以用无数种方式表达。"我很开心"、"心情愉悦"、"简直乐开了花"、"今天心情倍儿棒"表达的都是同一种情感，但字面完全不同。机器需要理解这些变体背后的一致语义。

**常识依赖**：人类对话中大量依赖隐含的背景知识。当有人说"房间里真冷"，人类知道这可能是请求关窗，而不是简单陈述温度。当有人说"我把苹果洗了"，人类知道这里说的是水果而不是手机。机器缺乏这种常识推理能力，这至今仍是 NLP 领域最具挑战性的问题之一。

## 2 NLP 发展历程

NLP 的发展可以清晰地划分为四个阶段。每个阶段都代表了一种根本性的范式转变。

### 第一阶段：规则时代（1950s-1980s）

1950 年，阿兰·图灵发表了著名论文《计算机器与智能》，提出了**图灵测试**——如果机器能在对话中让人无法分辨它是机器还是人类，就可以认为机器具有智能。这为 NLP 设定了宏伟目标。

这一时期由语言学家主导，核心思想是**用逻辑规则描述语言**。他们相信通过完备的语法和逻辑规则就能让计算机理解语言。代表性系统包括：

```text
1966 年 ELIZA：通过关键词匹配和句式重组模拟心理治疗师
  用户："我很不开心"
  ELIZA："你为什么说你很不开心？"

1970 年 SHRDLU：能在虚拟积木世界中理解并执行指令
  用户："把红色积木放到蓝色积木上面"
  系统：（执行操作）
```

但瓶颈很快暴露：语言的复杂性远超预期，规则难以穷尽，系统十分脆弱，无法处理规则之外的任何情况。一句不在规则中的表达就能让整个系统崩溃。

### 第二阶段：统计时代（1990s-2000s）

研究范式发生了重大转变。统计学派主张：**与其让专家告诉计算机规则，不如让计算机自己从数据中学习规律**。

核心思想是将语言现象的合理性转化为基于真实文本概率的数学计算。核心技术包括：

| 技术 | 类型 | 典型应用 |
|------|------|---------|
| N-gram | 统计语言模型 | 语音识别、拼写纠错 |
| HMM | 生成式序列模型 | 词性标注、分词 |
| CRF | 判别式序列模型 | NER、分词 |
| 统计机器翻译 | 对齐+翻译 | Google 翻译（2006） |

标志性成果是 2006 年上线的 **Google 翻译**，它基于统计机器翻译实现了远超规则系统的翻译质量，让大众首次切身享受到了高质量机器翻译的便利。

### 第三阶段：深度学习时代（2013-2017）

2013 年，**Word2Vec** 将词语表示为稠密的数字向量，使模型能在向量空间中捕捉语义规律。经典示例：

$$vector(\text{国王}) - vector(\text{男人}) + vector(\text{女人}) \approx vector(\text{女王})$$

2014 年，注意力机制在神经机器翻译中被系统化提出，用于在生成当前词时对输入序列的不同位置分配不同权重。2017 年，Google 团队发表了划时代的论文《Attention Is All You Need》，提出了 **Transformer** 架构，以自注意力为核心，摆脱了 RNN 的顺序计算限制，成为后续所有大模型的基础。

这一阶段的关键里程碑：

```text
2013 ── Word2Vec：词向量革命
2014 ── 注意力机制：突破信息瓶颈
2015 ── Seq2Seq + Attention：神经机器翻译
2017 ── Transformer：自注意力架构
```

### 第四阶段：大模型时代（2018-至今）

2018 年，**BERT** 通过掩码语言模型（Masked Language Model）学习上下文语义表示，推动了"预训练-微调"范式。同年，**GPT** 以自回归方式预训练，两种路线并行发展。

2020 年，**GPT-3** 以 1750 亿参数展现了惊人的少样本/零样本泛化能力——只需要给模型几个示例，它就能完成从未专门训练过的任务。这标志着 LLM 时代正式到来。

2022 年 11 月，**ChatGPT** 通过指令微调（Instruction Tuning）与人类反馈强化学习（RLHF）对齐，将大模型推向大众。两个月内用户突破 1 亿，成为史上增长最快的消费级应用。

```text
完整时间线：
1950 ── 图灵测试
1966 ── ELIZA 对话系统
1990s ── 统计方法兴起（HMM、CRF）
2006 ── Google 翻译上线
2013 ── Word2Vec（词向量革命）
2014 ── 注意力机制
2017 ── Transformer 架构
2018 ── BERT + GPT（预训练-微调范式）
2020 ── GPT-3（大模型时代）
2022 ── ChatGPT（AI 大众化）
2023-2024 ── GPT-4、Claude、LLaMA、开源 LLM 爆发
2025 ── 推理模型（o1、DeepSeek-R1）
```

## 3 NLP 核心任务分类

NLP 的任务种类繁多，但可以归纳为以下几个大类。理解这些任务有助于后续章节中理解 LLM 是如何"以一当十"地统一它们的。

### 3.1 文本分类

**文本分类**是将给定文本自动分配到一个或多个预定义类别中的任务。最典型的应用是**情感分析**和**主题分类**。

```text
输入："这部电影太棒了，强烈推荐！"
输出：正面情感

输入："NBA季后赛将于下周开始，湖人和勇士将在首轮对决。"
输出：体育类

输入："苹果公司发布了新款 MacBook。"
输出：科技类
```

文本分类的关键在于选择合适的特征表示和分类算法。传统方法使用 TF-IDF + SVM/朴素贝叶斯，现代方法则使用 BERT 等预训练模型进行微调。

### 3.2 序列标注

**序列标注**是为文本中的每个单元（字或词）分配一个标签。核心任务包括：

**命名实体识别（NER）**——从文本中找出并分类关键实体：

```text
输入：李雷和韩梅梅是北京市海淀区的居民。
输出：[(李雷, 人名), (韩梅梅, 人名), (北京市海淀区, 地名)]
```

**词性标注（POS Tagging）**——为每个词标注语法角色：

```text
输入：She is playing the guitar in the park.
输出：She(代词) is(动词) playing(动词) the(限定词) guitar(名词) in(介词) the(限定词) park(名词)
```

**关系抽取**——在识别实体的基础上判断实体间的语义关系：

```text
输入：比尔·盖茨是微软公司的创始人。
输出：创始人(比尔·盖茨, 微软公司)
```

序列标注是信息抽取的关键一步，将非结构化文本转为结构化信息，为知识图谱构建等下游任务提供基础。

### 3.3 序列生成

序列生成任务要求模型根据输入生成一段新的文本。核心任务包括：

**机器翻译**——将一种语言翻译成另一种：

```text
源语言：今天天气很好。
目标语言：The weather is very nice today.
```

机器翻译是 NLP 历史最悠久的任务之一，从最早的规则方法到统计方法，再到神经机器翻译（NMT），翻译质量不断提升。现代 LLM 的翻译能力已经接近甚至达到了专业译者的水平。

**文本摘要**——将长文本压缩为简短摘要。摘要可分为两大类：

```text
原文：2021年5月22日，国家航天局宣布，我国自主研发的火星探测器
"天问一号"成功在火星表面着陆。此次任务的成功，标志着我国在
深空探测领域迈出了重要一步。

抽取式摘要（直接选取关键句）：
  我国自主研发的火星探测器"天问一号"成功在火星表面着陆。

生成式摘要（理解后重新概括）：
  "天问一号"探测器成功实现火星着陆，代表我国在宇宙探索中
  取得重大进展。
```

**对话生成**——模拟多轮对话，理解上下文并作出恰当回应。这是 ChatGPT 等产品的核心能力。对话系统需要处理多轮上下文、保持一致性、理解隐含意图等复杂问题。

### 3.4 问答与阅读理解

**自动问答**使计算机能够理解自然语言问题并给出准确答案。可分为：

- **检索式问答**：从大量文档中检索答案，如搜索引擎
- **知识库问答**：从结构化知识库（如 Wikidata）中查询
- **生成式问答**：理解问题后直接生成答案，如 ChatGPT

```text
问题：珠穆朗玛峰多高？
答案：8848.86米

问题：Attention Is All You Need 的作者是谁？
答案：Ashish Vaswani, Noam Shazeer 等人
```

现代 LLM 的问答能力已经非常强大，但仍存在**幻觉问题**——模型有时会自信地编造不存在的事实。

### 3.5 信息检索

**信息检索（Information Retrieval, IR）**是根据用户查询从大量文档中找到最相关内容的任务。传统搜索引擎依赖 TF-IDF 等统计方法计算查询与文档的相关性，现代搜索引擎则越来越多地利用语义理解来提升检索质量。

近年来，**检索增强生成（Retrieval-Augmented Generation, RAG）**将信息检索与 LLM 结合，先从知识库中检索相关文档，再让 LLM 基于检索结果生成答案，有效缓解了幻觉问题。

## 4 文本表示的演进

文本表示是 NLP 的基础——如何将人类语言转化为计算机可以处理的数字形式。这个过程经历了多个阶段，每一步都带来了质的飞跃。

### 4.1 One-Hot 编码

最直观的方法是为每个词分配一个唯一的向量。假设词表大小为 6：

```text
"我"   -> [1, 0, 0, 0, 0, 0]
"喜欢" -> [0, 1, 0, 0, 0, 0]
"你"   -> [0, 0, 1, 0, 0, 0]
"苹果" -> [0, 0, 0, 1, 0, 0]
"香蕉" -> [0, 0, 0, 0, 1, 0]
"吃"   -> [0, 0, 0, 0, 0, 1]
```

**问题**：维度等于词表大小（通常数万），极其稀疏，稀疏率超过 99.97%；任意两个不同词的向量正交（点积为 0），完全无法表达语义相似性——"苹果"和"香蕉"的距离与"苹果"和"我"的距离完全相同。

### 4.2 TF-IDF

TF-IDF 通过词频和逆文档频率为每个词赋予权重，解决了"常见词权重过高"的问题。一个词的重要性与其在当前文档中的出现频率成正比，与在整个语料库中的出现频率成反比：

$$\text{TF-IDF}(t, d, D) = \text{TF}(t, d) \times \text{IDF}(t, D)$$

其中 IDF 的计算为：

$$\text{IDF}(t, D) = \log \frac{\vert D \vert}{1 + \vert \{d \in D : t \in d\} \vert}$$

TF-IDF 能有效衡量词的重要性，广泛用于关键词提取和文档相似度计算。但它仍然是稀疏表示，且无法捕捉语义关系。

### 4.3 词向量（Word2Vec）

2013 年 Word2Vec 将词映射为低维（通常 100-300 维）的稠密向量，语义相近的词在向量空间中距离更近。这解决了稀疏性和语义鸿沟两大问题。但它是**静态**的——每个词只有一个固定向量，无法处理一词多义。

### 4.4 上下文表示

ELMo（2018）首次引入**动态词向量**，同一个词在不同上下文中会有不同的向量表示。"苹果"在"吃苹果"和"苹果手机"中的表示完全不同。

BERT（2018）通过 Transformer 的自注意力机制，进一步提升了上下文表示的质量。每个 token 的表示都融合了双向上下文信息。

### 4.5 LLM 时代

大语言模型不再显式地学习"词向量"，而是通过海量文本上的自回归预训练，在模型内部隐式地学习了极其丰富的语言表示。一个 GPT-4 级别的模型，其内部表示已经远远超越了传统词向量所能表达的语义范围。

```text
演进路线：
One-Hot (稀疏、无语义)
  → TF-IDF (加权、仍是稀疏)
    → Word2Vec (稠密、静态)
      → ELMo/BERT (稠密、动态上下文)
        → LLM (隐式、超大规模语义)
```

## 5 为什么 LLM 改变了一切

大语言模型对 NLP 领域的冲击是颠覆性的，主要体现在两个根本性转变。

### 从任务特定到通用基座

传统 NLP 范式是**一个任务一个模型**：情感分析用一个模型，NER 用另一个模型，翻译再用一个模型。每个模型都需要专门的标注数据、特征工程和训练流程。

```text
传统范式：
  情感分析 ──→ 收集标注数据 ──→ 特征工程 ──→ 训练模型 A
  命名实体识别 → 收集标注数据 ──→ 特征工程 ──→ 训练模型 B
  机器翻译 ──→ 收集平行语料 ──→ 对齐特征 ──→ 训练模型 C

LLM 范式：
  通用大模型（一次训练） + Prompt ──→ 完成 A、B、C 及更多任务
```

LLM 通过在海量文本上预训练，学习了通用的语言能力。在实际使用时，只需通过适当的提示（Prompt）就能让同一个模型完成各种不同的任务。这被称为**基础模型（Foundation Model）**范式。

### 从特征工程到 Prompt Engineering

传统 NLP 需要大量人工设计特征（如 TF-IDF、n-gram、词性标注等），这个过程称为**特征工程**，需要深厚的领域专业知识。一个优秀的 NLP 工程师需要花大量时间在特征设计上。

LLM 时代，核心技能变成了 **Prompt Engineering**——通过设计合适的提示词来引导模型完成任务。例如：

```text
任务：判断以下评论的情感

Prompt：
请判断以下电影评论的情感倾向（正面/负面）：
"这部电影太棒了，强烈推荐！"

LLM 输出：正面
```

这大大降低了 NLP 的应用门槛。不再需要收集大量标注数据、设计复杂特征、训练专门模型——一个 Prompt 就够了。

### 但 LLM 不是万能的

尽管 LLM 带来了革命性进步，但它也面临独特挑战：

- **幻觉问题**：模型有时会以极高的自信编造事实，这在医疗、法律等高风险场景中是致命的
- **推理局限**：在复杂逻辑推理和数学证明上仍有不足
- **可控性**：难以精确控制输出内容的格式、风格和事实准确性
- **成本**：训练和部署大模型需要巨大的计算资源和能源
- **可解释性**：我们难以理解模型为什么做出某个特定决策
- **数据偏见**：训练数据中的偏见可能被模型内化甚至放大

## 7 NLP 面临的主要挑战

除了前文提到的语言本身的挑战（歧义、多样性、常识依赖），NLP 在工程和伦理层面也面临诸多挑战：

**模型幻觉**：大语言模型有时会以极高的自信编造事实，这种"一本正经地胡说八道"严重影响了它在新闻、法律、科研等严谨场景下的可靠性。

**数据质量与偏见**：全球数千种语言中，只有英语、中文等少数语言拥有海量高质量数据，绝大多数语言属于低资源语言。另一方面，训练数据中潜藏的性别歧视、种族偏见一旦被模型内化甚至放大，会导致算法产出不公平的观点。

**计算成本**：训练和部署顶尖大模型需要庞大的算力集群和惊人的能源消耗。GPT-4 的训练成本估计超过 1 亿美元，这构成了极高的经济门槛。

**可解释性**：深度学习模型的"黑箱"性质使我们难以知晓模型做出某个特定决策的具体依据，这在医疗诊断、金融风控等高风险领域是巨大的阻碍。

## 8 常见误区

**误区 1："NLP 就是做聊天机器人"**

NLP 的范围远不止对话系统。文本分类、信息抽取、机器翻译、文本摘要、问答系统等都是 NLP 的核心任务。聊天机器人只是众多应用之一。NLP 的目标是让计算机理解和处理人类语言的所有方面。

**误区 2："有了 LLM 就不需要学传统 NLP 了"**

理解传统方法（如分词、TF-IDF、N-gram）有助于理解 LLM 的设计动机和局限性。很多传统思想直接启发了现代 LLM——例如注意力机制的前身就是统计对齐，Transformer 中的位置编码与 N-gram 捕捉词序的思想一脉相承。而且，在资源受限的场景中，传统方法仍然有其实用价值。

**误区 3："中文分词和英文分词是一样的"**

中文没有天然的空格分隔符，分词是中文 NLP 特有的重要步骤。英文虽然相对简单，但也有缩写（don't）、复合词（New York）等需要处理的情况。现代 LLM 使用的子词分词（BPE）则是一种跨语言的通用方案，在很大程度上统一了不同语言的分词策略。

**误区 4："词向量越大越好"**

词向量维度是超参数，需要在表达能力和计算效率之间权衡。通常 100-300 维对 Word2Vec 就足够了。更大的维度不一定带来更好的效果，反而可能过拟合。

**误区 5："NLP 的终极目标是通过图灵测试"**

图灵测试是一个历史概念，现代 NLP 的目标更加务实——在具体任务上提供有价值的工具和服务，而不是追求"像人一样思考"。事实上，当前的 LLM 在很多任务上已经超越了人类水平，但仍然缺乏真正的理解能力。

## 9 小结

本章从宏观视角介绍了 NLP 的全貌。核心要点如下：

1. NLP 的目标是让计算机理解、解释和生成人类语言，面临歧义性、多样性和常识依赖三大挑战
2. NLP 经历了规则→统计→深度学习→大模型四个阶段，每个阶段都代表了范式的根本转变
3. 核心任务包括文本分类、序列标注、序列生成、问答和信息检索
4. 文本表示从 One-Hot 到 TF-IDF 到词向量到上下文表示，逐步解决了稀疏性和语义鸿沟
5. LLM 带来了从任务特定到通用基座、从特征工程到 Prompt Engineering 的范式转变
