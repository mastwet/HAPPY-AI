# 28 注意力机制 ⭐

上一章我们看到了 Seq2Seq 架构的核心缺陷——**信息瓶颈**。编码器将整个输入序列压缩成一个固定长度的向量，解码器在每一步都使用同一个向量，无法有选择性地关注输入的不同部分。

本章将介绍彻底改变这一切的技术——**注意力机制（Attention Mechanism）**。它不仅解决了 Seq2Seq 的信息瓶颈，更成为 Transformer 架构的核心组件，是当今所有大语言模型（GPT、LLaMA、Qwen 等）的基石。

## 1 直觉：人类的注意力

![注意力机制总览](/images/ch28_1-0.png)


想象你在看一张照片。你的眼睛并不会同时"看清"画面的每一个像素——你会自然地将注意力聚焦在关键区域：人脸、文字、重要物体。当你需要回答"照片里有几个人？"时，你的注意力会集中在人脸区域；当回答"背景是什么颜色？"时，注意力会转移到背景区域。

注意力机制就是对这种认知行为的数学模拟。它让模型在处理序列的每一步，都能**动态地决定关注输入的哪些部分**，而不是被迫将所有信息压缩成一个固定表示。

## 2 注意力机制的起源

### 2.1 Bahdanau Attention（2014）

注意力机制由 Bahdanau 等人在 2014 年的论文《Neural Machine Translation by Jointly Learning to Align and Translate》中系统化提出 [1]。他们的动机非常直接：标准 Seq2Seq 在翻译长句子时性能急剧下降，因为固定长度的上下文向量无法承载所有信息。

核心思想：**解码器在生成每个词时，不再依赖一个固定的上下文向量，而是动态地计算一个专属当前时间步的上下文向量 $C_t$**。

### 2.2 动态加权原理

具体做法是：让解码器在每一步"回头看"编码器的所有隐藏状态，并根据当前需求分配不同的权重：

$$C_t = \sum_{j=1}^{T_x} \alpha_{tj} h_j$$

其中 $\alpha_{tj}$ 是解码器在生成第 $t$ 个词时，对编码器第 $j$ 个位置的**注意力权重**。这组权重通过 Softmax 归一化，总和为 1。

### 2.3 注意力权重的计算

注意力权重通过三步计算得到：

**第一步：计算相似度分数**

使用解码器上一时刻的状态 $h'_{t-1}$ 与编码器的每个状态 $h_j$ 计算分数：

$$e_{tj} = \text{score}(h'_{t-1}, h_j)$$

**第二步：归一化**

用 Softmax 将分数转为权重：

$$\alpha_{tj} = \frac{\exp(e_{tj})}{\sum_{i=1}^{T_x} \exp(e_{ti})}$$

**第三步：加权求和**

用权重对编码器状态加权求和，得到动态上下文向量 $C_t$。

这个机制让模型在翻译 "cat" 时可以重点关注源句子中 "cat" 对应的位置，在翻译 "tired" 时重点关注 "tired" 对应的位置——这就是"注意力"的含义。

## 3 Query-Key-Value 框架

![QKV 框架示意图](/images/ch28_1-1.png)


### 3.1 图书馆检索类比

为了将注意力计算抽象成通用框架，我们引入 **Query-Key-Value（QKV）** 概念。可以用图书馆检索来类比：

```text
你想找关于"深度学习"的书（Query）
图书馆里每本书都有一个标签（Key）：["机器学习", "深度学习", "烹饪", "历史"]
每本书都有实际内容（Value）：[书1的内容, 书2的内容, 书3的内容, 书4的内容]

注意力计算过程：
1. 用你的 Query 和每个 Key 比较相似度 → [0.3, 0.9, 0.05, 0.01]
2. Softmax 归一化 → [0.2, 0.65, 0.1, 0.05]
3. 用权重对 Value 加权求和 → 0.2×书1 + 0.65×书2 + 0.1×书3 + 0.05×书4
```

最终你得到的是一份以"深度学习"相关内容为主的综合信息。

### 3.2 Q/K/V 在 Seq2Seq 中的含义

在 Seq2Seq 的注意力中：

```text
Query（查询）：解码器当前状态 h'_{t-1}，代表"我现在需要什么信息"
Key（键）：    编码器各时刻的隐藏状态 h_j，代表"这里有什么信息的标签"
Value（值）：  编码器各时刻的隐藏状态 h_j，代表"实际的信息内容"
```

在基础注意力中，Key 和 Value 是相同的（都来自编码器状态）。但在更复杂的模型中，它们可以不同——Q、K、V 各自经过独立的线性变换，让模型在不同的"子空间"中进行匹配和聚合。

### 3.3 为什么要分开 Q 和 K

一个自然的问题：既然 Key 和 Value 通常相同，为什么不直接用一个概念？

原因是 Q 和 K 的关系是**非对称的**。"我在找什么"（Query）和"这里有什么"（Key）是两个不同的角色。分开它们可以让模型学到更灵活的匹配模式：

- 在自注意力中，Q 和 K 来自同一个序列，但经过不同的线性变换，捕捉不同的语义角色
- 在交叉注意力中，Q 来自解码器，K 来自编码器，它们的语义空间完全不同
- 分开 Q 和 K 还允许非对称的相似度计算（$q_i$ 对 $k_j$ 的关注程度 ≠ $k_j$ 对 $q_i$ 的关注程度）

## 4 Scaled Dot-Product Attention

### 4.1 数学推导

注意力机制最核心的计算公式是**缩放点积注意力（Scaled Dot-Product Attention）**，由 Vaswani 等人在 2017 年的 Transformer 论文中提出 [2]：

$$\text{Attention}(Q,K,V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

其中：
- $Q$ 是查询矩阵，形状为 $(n, d_k)$
- $K$ 是键矩阵，形状为 $(m, d_k)$
- $V$ 是值矩阵，形状为 $(m, d_v)$
- $d_k$ 是键向量的维度
- 输出形状为 $(n, d_v)$

让我们逐步拆解这个公式：

**Step 1：$QK^T$——计算注意力分数**

$QK^T$ 的结果是一个 $(n, m)$ 的矩阵，其中第 $i$ 行第 $j$ 列的元素是 $q_i$ 和 $k_j$ 的点积。点积是衡量向量相似度的有效方式——两个向量越相似，点积越大。

**Step 2：$\frac{QK^T}{\sqrt{d_k}}$——缩放**

这是关键的一步。为什么要除以 $\sqrt{d_k}$？

### 4.2 为什么要除以 $\sqrt{d_k}$

假设 $q$ 和 $k$ 的每个维度都是独立的、均值为 0、方差为 1 的随机变量。那么它们的点积：

$$q \cdot k = \sum_{i=1}^{d_k} q_i k_i$$

根据独立随机变量的性质，点积的均值为 0，**方差为 $d_k$**。当 $d_k$ 很大时（如 64、128），点积的值会非常大。

大的点积值会导致 Softmax 函数进入**饱和区**——输出极端地接近 0 或 1，梯度接近 0，模型难以训练。

```text
不缩放时（d_k = 128）：
  点积值可能为 [-30, +30]
  Softmax 输出：[0.0001, 0.0001, ..., 0.9999, ...]
  梯度几乎为 0 → 训练困难

缩放后（除以 sqrt(128) ≈ 11.3）：
  点积值缩放到 [-2.7, +2.7]
  Softmax 输出：[0.02, 0.05, ..., 0.3, ...]
  梯度适中 → 训练稳定
```

除以 $\sqrt{d_k}$ 使得无论维度多大，点积的方差都保持为 1，确保 Softmax 不会饱和。

**Step 3：$\text{softmax}(\cdot)$——归一化**

将分数转为概率分布，所有权重非负且和为 1。

**Step 4：$\cdot V$——加权求和**

用注意力权重对 Value 矩阵加权求和，得到最终输出。

## 5 注意力的 PyTorch 实现

![注意力计算流程](/images/ch28_1-2.png)


### 5.1 基础版本

```python
import torch
import torch.nn as nn
import math

def scaled_dot_product_attention(query, key, value, mask=None):
    """
    Args:
        query: (batch, n, d_k) - 查询矩阵
        key:   (batch, m, d_k) - 键矩阵
        value: (batch, m, d_v) - 值矩阵
        mask:  (batch, n, m)   - 可选的掩码矩阵
    Returns:
        output: (batch, n, d_v) - 注意力输出
        weights: (batch, n, m)  - 注意力权重
    """
    d_k = query.size(-1)

    # Step 1-2: 计算缩放点积注意力分数
    scores = torch.matmul(query, key.transpose(-2, -1)) / math.sqrt(d_k)

    # Step 3: 应用掩码（如果有的话）
    if mask is not None:
        scores = scores.masked_fill(mask == 0, float("-inf"))

    # Step 4: Softmax 归一化
    weights = torch.softmax(scores, dim=-1)

    # Step 5: 加权求和
    output = torch.matmul(weights, value)

    return output, weights
```

### 5.2 带线性投影的版本

在实际模型中，Q、K、V 通常需要经过线性变换后再进行注意力计算：

```python
class Attention(nn.Module):
    def __init__(self, d_model, d_k, d_v):
        super().__init__()
        self.W_q = nn.Linear(d_model, d_k, bias=False)
        self.W_k = nn.Linear(d_model, d_k, bias=False)
        self.W_v = nn.Linear(d_model, d_v, bias=False)

    def forward(self, query, key, value, mask=None):
        # 线性投影
        Q = self.W_q(query)
        K = self.W_k(key)
        V = self.W_v(value)

        # 缩放点积注意力
        d_k = Q.size(-1)
        scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(d_k)

        if mask is not None:
            scores = scores.masked_fill(mask == 0, float("-inf"))

        weights = torch.softmax(scores, dim=-1)
        output = torch.matmul(weights, value)

        return output, weights
```

线性投影的作用是让模型学习在不同的"子空间"中进行匹配。例如，$W_q$ 可以学习将输入投影到"需求空间"，$W_k$ 投影到"标签空间"，使得匹配更加精准。

## 6 自注意力（Self-Attention）

### 6.1 核心思想

在 Seq2Seq 的注意力中，Q 来自解码器，K/V 来自编码器——这是两个不同序列之间的**交叉注意力**。**自注意力（Self-Attention）** 则是让 Q、K、V 全部来自**同一个输入序列**：

$$\text{Self-Attention}: Q = K = V = X$$

它让序列中的每个位置都能关注到序列中的所有其他位置，从而捕捉序列内部的依赖关系。

```text
输入序列："The cat sat on the mat"

自注意力让每个词都能"看到"其他所有词：
  "cat" → 高度关注 "The" 和 "sat"（主语和谓语的关系）
  "sat" → 关注 "cat"（谁坐了）和 "mat"（坐在哪）
  "mat" → 关注 "the" 和 "on"（限定词和介词）
```

### 6.2 与 RNN 的对比

自注意力相比 RNN 有两个关键优势：

**并行计算**：RNN 必须按顺序逐个处理 token，自注意力可以一次性计算所有位置之间的关系，充分利用 GPU 并行能力。

**直接连接**：RNN 中距离为 $n$ 的两个 token 之间需要经过 $n$ 步传递，信息会衰减。自注意力中任意两个 token 之间都是**直接连接**，距离为 1。

```text
RNN 的信息路径（"cat" 到 "mat"）：
  cat → h_2 → h_3 → h_4 → h_5 → mat  （经过 4 步传递）

自注意力的信息路径：
  cat → mat  （直接连接，一步到位）
```

### 6.3 自注意力的直觉

可以将自注意力理解为一种**投票机制**。序列中的每个 token 都有一张"选票"（Query），同时每个 token 也都是一个"候选人"（Key）。投票时，每个 token 会根据与候选人的"亲和度"分配票数，最终根据得票数对 Value 进行加权汇总。

## 7 掩码自注意力（Masked Self-Attention）

### 7.1 为什么需要掩码

在语言模型的训练中，模型需要预测序列中的下一个 token。例如，对于句子 "I love you"：

```text
Step 1：看到 [I] → 预测 "love"
Step 2：看到 [I, love] → 预测 "you"
Step 3：看到 [I, love, you] → 预测 "<EOS>"
```

如果使用普通的自注意力，"love" 在计算注意力时会"看到"后面的 "you"——这等于**偷看了答案**。掩码自注意力通过将未来位置的注意力分数设为 $-\infty$，确保每个位置只能关注它自己和之前的位置。

### 7.2 因果掩码的实现

因果掩码（Causal Mask）是一个上三角矩阵，上三角位置为 $-\infty$，下三角和对角线为 0：

```text
序列长度为 4 的因果掩码：
  [[  0,    -inf, -inf, -inf],
   [  0,      0,  -inf, -inf],
   [  0,      0,    0,  -inf],
   [  0,      0,    0,     0]]
```

在计算 Softmax 之前加上这个掩码，$-\infty$ 的位置经过 Softmax 后会变成 0，从而屏蔽了未来信息。

```python
def create_causal_mask(seq_len):
    """创建因果掩码（下三角为1，上三角为0）"""
    mask = torch.tril(torch.ones(seq_len, seq_len))
    return mask.unsqueeze(0)  # 增加 batch 维度

# 使用示例
seq_len = 4
mask = create_causal_mask(seq_len)
scores = torch.randn(1, seq_len, seq_len)
masked_scores = scores.masked_fill(mask == 0, float("-inf"))
attention_weights = torch.softmax(masked_scores, dim=-1)
```

### 7.3 GPT 的核心机制

掩码自注意力是 GPT 系列模型的核心。GPT 采用**Decoder-only** 架构，整个模型就是一个巨大的掩码自注意力网络。它通过预测下一个 token 的任务在海量文本上预训练，学习语言的统计规律。

```text
GPT 的训练过程：
  输入："今天 天气 真 好"
  模型内部使用掩码自注意力：
    位置 1 "今天" → 只能看到 "今天" → 预测 "天气"
    位置 2 "天气" → 能看到 "今天 天气" → 预测 "真"
    位置 3 "真"   → 能看到 "今天 天气 真" → 预测 "好"
    位置 4 "好"   → 能看到全部 → 预测 "<EOS>"
```

这种设计使得训练时可以**并行处理整个序列**（不像 RNN 必须串行），同时通过掩码保证了因果性——每个位置的预测只依赖于之前的 token。

## 8 交叉注意力（Cross-Attention）

### 8.1 Encoder-Decoder 的桥梁

交叉注意力（Cross-Attention）是连接编码器和解码器的桥梁。与自注意力不同，它的 Q 来自解码器，K 和 V 来自编码器：

$$\text{Cross-Attention}: Q = \text{Decoder State}, \quad K = V = \text{Encoder Outputs}$$

```text
编码器输出：[h_1, h_2, h_3, ..., h_T]  (源序列的表示)
解码器状态：s_{t-1}                      (当前生成位置)

交叉注意力：
  Q = s_{t-1}（"我现在需要什么信息"）
  K = [h_1, h_2, ..., h_T]（"源序列各位置有什么"）
  V = [h_1, h_2, ..., h_T]（"源序列各位置的实际内容"）

输出：一个聚焦于源序列相关部分的上下文向量
```

### 8.2 解决信息瓶颈

交叉注意力彻底解决了 Seq2Seq 的信息瓶颈问题：

```text
标准 Seq2Seq：
  编码器 → [C] → 解码器（每步用同一个 C）
  瓶颈：所有信息必须通过 C 这个"独木桥"

带交叉注意力的 Seq2Seq：
  编码器 → [h_1, h_2, ..., h_T] → 解码器（每步动态计算 C_t）
  无瓶颈：解码器可以直接访问编码器的每一步输出
```

解码器在生成每个词时，都可以根据当前需求，动态地从编码器的所有输出中提取最相关的信息。

### 8.3 交叉注意力在现代 LLM 中的应用

交叉注意力不仅用于 Seq2Seq 翻译模型，还在多种场景中发挥作用：

- **多模态模型**：文本 Q 关注图像 K/V（如 Flamingo、LLaVA 的某些变体）
- **检索增强生成（RAG）**：生成时关注检索到的文档
- **Encoder-Decoder LLM**：T5、BART 等模型在解码器中使用交叉注意力关注编码器输出

## 9 多头注意力（Multi-Head Attention）

![多头注意力机制](/images/ch28_1-3.jpeg)


### 9.1 为什么需要多头

一次注意力计算只能捕捉一种相关性。但语言中的关系是多层次的：

```text
"The cat sat on the mat because it was tired"

"it" 需要同时理解：
  - 指代关系：it → cat（语法层面）
  - 因果关系：because（逻辑层面）
  - 空间关系：on the mat（语义层面）
```

单次注意力计算很难同时捕捉所有这些关系。**多头注意力**的解决方案是：**并行执行多次注意力计算，每次关注不同类型的关系，最后将结果拼接起来**。

### 9.2 数学形式

$$\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \dots, \text{head}_h)W^O$$

其中每个头独立计算：

$$\text{head}_i = \text{Attention}(QW_i^Q, KW_i^K, VW_i^V)$$

参数说明：
- $h$ 是头的数量（如 8、12、32）
- $W_i^Q \in \mathbb{R}^{d_{\text{model}} \times d_k}$，$W_i^K \in \mathbb{R}^{d_{\text{model}} \times d_k}$，$W_i^V \in \mathbb{R}^{d_{\text{model}} \times d_v}$
- 通常 $d_k = d_v = d_{\text{model}} / h$
- $W^O \in \mathbb{R}^{hd_v \times d_{\text{model}}}$ 是输出投影矩阵

### 9.3 实现

```python
class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, n_heads):
        super().__init__()
        assert d_model % n_heads == 0

        self.d_model = d_model
        self.n_heads = n_heads
        self.d_k = d_model // n_heads

        # QKV 投影矩阵
        self.W_q = nn.Linear(d_model, d_model, bias=False)
        self.W_k = nn.Linear(d_model, d_model, bias=False)
        self.W_v = nn.Linear(d_model, d_model, bias=False)

        # 输出投影
        self.W_o = nn.Linear(d_model, d_model, bias=False)

    def forward(self, query, key, value, mask=None):
        batch_size = query.size(0)

        # 1. 线性投影
        Q = self.W_q(query)  # (batch, seq, d_model)
        K = self.W_k(key)
        V = self.W_v(value)

        # 2. 拆分成多头: (batch, seq, d_model) -> (batch, n_heads, seq, d_k)
        Q = Q.view(batch_size, -1, self.n_heads, self.d_k).transpose(1, 2)
        K = K.view(batch_size, -1, self.n_heads, self.d_k).transpose(1, 2)
        V = V.view(batch_size, -1, self.n_heads, self.d_k).transpose(1, 2)

        # 3. 缩放点积注意力
        scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(self.d_k)

        if mask is not None:
            scores = scores.masked_fill(mask == 0, float("-inf"))

        weights = torch.softmax(scores, dim=-1)
        context = torch.matmul(weights, V)

        # 4. 拼接多头: (batch, n_heads, seq, d_k) -> (batch, seq, d_model)
        context = context.transpose(1, 2).contiguous().view(batch_size, -1, self.d_model)

        # 5. 输出投影
        output = self.W_o(context)

        return output, weights
```

### 9.4 参数量分析

以 $d_{\text{model}} = 512$，$h = 8$ 为例：

```text
每个头的维度：d_k = d_v = 512 / 8 = 64

QKV 投影参数：3 × (512 × 64) × 8 = 786,432
  等价于：3 × (512 × 512) = 786,432（因为 64 × 8 = 512）

输出投影参数：512 × 512 = 262,144

总参数：786,432 + 262,144 = 1,048,576 ≈ 1M
```

多头注意力的参数量与单头注意力相同（都是 $4 \times d_{\text{model}}^2$），但表达能力更强——它在不同的子空间中并行学习不同的注意力模式。

### 9.5 多头注意力的可视化

通过可视化不同头的注意力权重，可以观察到每个头学会了不同的语言学模式：

```text
Head 1：学会了语法依赖（主语-谓语、动词-宾语）
  "The cat sat" → cat 和 sat 之间有强连接

Head 2：学会了指代关系（代词-先行词）
  "it was tired" → it 和 cat 之间有强连接

Head 3：学会了相邻词关系
  每个词主要关注它的前一个词

Head 4：学会了位置模式
  关注句子开头或结尾的特定位置
```

这种分工是模型自动学习的，不需要人工设计。多头注意力的威力正在于此——它让模型自动发现并利用语言中多种层次的结构。

## 10 注意力与 Seq2Seq 的对比

### 10.1 信息瓶颈的解决

```text
标准 Seq2Seq：
  编码器 → [C] → 解码器
  C 是固定长度向量，承载整个序列的信息
  生成每个词时使用同一个 C
  → 信息瓶颈 + 无差别对待

带注意力的 Seq2Seq：
  编码器 → [h_1, h_2, ..., h_T] → 解码器
  解码器每步动态计算 C_t = sum(alpha_tj * h_j)
  不同的词关注不同的输入位置
  → 无瓶颈 + 动态聚焦
```

### 10.2 可解释性

注意力权重提供了天然的**可解释性**。通过可视化注意力矩阵，可以看到模型在生成每个输出词时"关注"了输入的哪些位置。这在机器翻译中表现为**词对齐**——一种自动学习的、无需人工标注的源-目标词对应关系。

### 10.3 从注意力到 Transformer

注意力机制的成功揭示了一个关键洞察：**直接建立序列中任意两个位置之间的连接，比通过 RNN 的循环传递更高效、更强大**。

这直接催生了 2017 年的 Transformer 架构——一个完全基于注意力、完全抛弃 RNN 的模型。Transformer 用**自注意力**替代了 RNN 的循环计算，实现了真正的并行化，并彻底解决了长距离依赖问题。

```text
RNN → Seq2Seq → 注意力机制 → Transformer → GPT/BERT/LLaMA
                                        ↑
                                    本章的核心贡献
```

## 11 常见误区

**误区 1："注意力机制就是 Transformer"**

注意力机制早在 2014 年就被提出（Bahdanau Attention），最初是作为 RNN Seq2Seq 的增强组件。Transformer（2017）的创新在于**完全用注意力替代了 RNN**，并引入了自注意力、多头注意力等关键设计。

**误区 2："除以 $\sqrt{d_k}$ 只是一个小技巧"**

这个缩放步骤对训练稳定性至关重要。不缩放时，当维度较大，Softmax 会进入饱和区，梯度接近 0，模型几乎无法训练。这不是"锦上添花"，而是"不可或缺"。

**误区 3："多头注意力的每个头学的东西完全不同"**

虽然不同头确实倾向于学习不同的模式，但它们之间也有大量的重叠和冗余。研究发现，某些头可能学到了相似的模式，甚至有些头可以被剪枝掉而不影响性能。

**误区 4："注意力权重代表了模型的'解释'"**

注意力权重提供了有价值的可解释性线索，但它们不一定是因果解释。模型的最终决策可能受到多层注意力的复杂影响，单层的注意力权重不能完全代表模型的推理过程。

**误区 5："自注意力的计算复杂度是 $O(n^2)$，所以不能处理长序列"**

这是事实（$n$ 是序列长度），但有很多优化方案：稀疏注意力、线性注意力、Flash Attention 等。现代 LLM 通过这些技术可以处理数十万甚至百万 token 的序列。

## 小结

本章介绍了注意力机制的核心内容：

1. **注意力的本质**是动态加权——根据当前需求，对输入的不同部分分配不同的权重
2. **QKV 框架**将注意力抽象为通用的检索模式：Query 查找最相关的 Key，提取对应的 Value
3. **Scaled Dot-Product Attention** 通过除以 $\sqrt{d_k}$ 防止 Softmax 饱和，是 Transformer 的核心计算
4. **自注意力**让序列中的每个位置关注所有其他位置，是 Transformer Encoder 的基础
5. **掩码自注意力**通过因果掩码保证只看到过去的信息，是 GPT 等 Decoder-only 模型的核心
6. **交叉注意力**连接编码器和解码器，彻底解决了 Seq2Seq 的信息瓶颈
7. **多头注意力**在不同的子空间中并行学习多种注意力模式，大幅提升模型表达能力

注意力机制不仅解决了 Seq2Seq 的信息瓶颈，更开启了"Attention Is All You Need"的时代——下一章我们将学习 Transformer 的完整架构，看看注意力是如何成为深度学习的核心构建块的。
