# 30 预训练模型：BERT、GPT、T5 与 HuggingFace 生态

在 Transformer 架构问世之后，NLP 领域迎来了一场范式革命。研究者们不再为每个任务从零训练模型，而是先在海量无监督语料上**预训练**一个通用的语言表示模型，再通过少量有监督数据**微调**到具体任务。这就是"预训练 + 微调"范式——它深刻改变了 NLP 的研究与工程实践。

本章将系统介绍三种代表性的预训练模型架构：BERT（Encoder-only）、GPT（Decoder-only）和 T5（Encoder-Decoder），并展示如何通过 HuggingFace 生态快速使用这些模型。

---

## 30.1 预训练的动机：从任务特定到通用表示

在预训练范式出现之前，NLP 的主流做法是为每个任务设计专用模型结构，用有监督数据从零训练。这种方式存在两个核心问题：

1. **数据效率低**：每个任务都需要大量标注数据，而标注成本高昂。
2. **知识无法复用**：模型在任务 A 上学到的语言知识，无法迁移到任务 B。

**预训练**的核心思想是：先让模型在海量无标注文本上学习通用的语言表示（语法、语义、世界知识），再将这些知识迁移到各种下游任务。这就像一个博览群书的学生，先积累广泛的知识基础，再针对具体考试做针对性练习——后者所需的努力远小于从零学起。

预训练范式的关键优势在于：

- **数据杠杆**：无标注文本几乎无限（互联网语料），预训练可以利用数 TB 甚至数十 TB 的数据。
- **一次训练，多次复用**：一次昂贵的预训练，可以服务于数百个下游任务。
- **涌现能力**：当模型规模和数据量达到一定程度时，模型会表现出训练目标未明确指定的能力（这将在第 31 章详细讨论）。

## 30.2 BERT

![BERT 模型架构](/images/ch30_5_1_1.png)
：双向编码的里程碑

**BERT**（Bidirectional Encoder Representations from Transformers）由 Google 于 2018 年发布，是预训练语言模型时代的开创性工作。它首次在 11 个 NLP 评测任务上同时取得 SOTA，标志着"预训练 + 微调"范式的正式确立。

### 30.2.1 架构：Encoder-Only

BERT 只使用 Transformer 的 **Encoder** 部分进行堆叠。其核心特点是**双向注意力**——在计算每个 token 的表示时，模型可以同时看到它左边和右边的所有 token。这与 GPT 的单向注意力形成鲜明对比。

BERT 提供了两种规模：

| 模型 | 层数 (L) | 隐藏维度 (H) | 注意力头数 | 参数量 |
|------|---------|-------------|-----------|--------|
| BERT-Base | 12 | 768 | 12 | 110M |
| BERT-Large | 24 | 1024 | 16 | 340M |

**输入表示**由三部分相加而成：

$$\text{Input}_{\text{embedding}} = \text{Token}_{\text{embedding}} + \text{Position}_{\text{embedding}} + \text{Segment}_{\text{embedding}}$$

- **Token Embedding**：使用 WordPiece 分词，将词拆分为子词（如 "playing" → ["play", "##ing"]）。
- **Position Embedding**：可学习的绝对位置编码（而非 Transformer 原始的正余弦编码），这也是 BERT 最大输入长度为 512 token 的原因。
- **Segment Embedding**：用于区分句子对中的两个句子（句子 A 和句子 B）。

BERT 还引入了几个特殊 token：`[CLS]`（序列开头，其输出用于分类任务）、`[SEP]`（分隔句子）、`[MASK]`（预训练时的掩码标记）。

### 30.2.2 预训练任务一：MLM（掩码语言模型）

传统的语言模型（如 GPT）是单向的，只能根据上文预测下文。BERT 提出了一种全新的预训练任务——**掩码语言模型（Masked Language Model, MLM）**，可以理解为"完形填空"：

1. 随机选择输入中 **15%** 的 token。
2. 对选中的 token 采用 **80/10/10 策略**：
   - 80% 的概率替换为 `[MASK]`
   - 10% 的概率替换为随机 token
   - 10% 的概率保持不变

这种策略的设计精妙之处在于：10% 保持不变缓解了预训练（有 `[MASK]`）和微调（无 `[MASK]`）之间的不一致；10% 随机替换迫使模型对每个 token 都保持上下文表征，而非只关注被遮蔽的位置。

MLM 让 BERT 能够同时利用上下文信息，从而学习到**深层双向语义关系**——这是 BERT 在自然语言理解任务上表现优异的关键。

### 30.2.3 预训练任务二：NSP（下一句预测）

为了学习句子之间的关系，BERT 还提出了**下一句预测（Next Sentence Prediction, NSP）**任务：给模型输入两个句子，让模型判断第二个句子是否是第一个句子的真正下一句。这是一个二分类任务，使用 `[CLS]` token 的输出进行预测。

NSP 的训练数据可以从任何文本中自动生成：正样本是连续的两个句子，负样本是随机拼接的两个句子。这使得预训练数据几乎是无限的。

> **NSP 的争议**：后续研究（如 RoBERTa、ALBERT）发现 NSP 任务过于简单，对下游任务帮助有限甚至有害。RoBERTa 直接移除了 NSP，ALBERT 则提出了更难的 SOP（句子顺序预测）来替代。

### 30.2.4 微调范式

预训练完成后，BERT 可以通过简单地添加任务特定的输出层来适配各种下游任务：

- **文本分类**：取 `[CLS]` 的输出，接一个全连接层。
- **序列标注**（如命名实体识别）：取每个 token 的输出，接分类层。
- **问答**：取段落中每个 token 的输出，预测答案的起止位置。

微调时使用较小的学习率（通常 $2 \times 10^{-5}$ 到 $5 \times 10^{-5}$），在任务数据上训练几个 epoch 即可。由于模型已经具备强大的语言理解能力，微调通常只需要几百到几千条标注数据。

### 30.2.5 BERT 的局限

BERT 虽然强大，但也存在明显的局限：

- **不能自回归生成**：BERT 的双向注意力机制使其天然不适合文本生成任务。它无法像 GPT 那样逐 token 生成文本。
- **最大长度限制**：由于使用可学习的绝对位置编码，BERT 的最大输入长度固定为 512 token，无法处理更长的文本。
- **预训练与微调不一致**：MLM 中的 `[MASK]` token 在微调和推理时不存在，导致一定的性能损失。

尽管如此，BERT 在标注数据丰富的 NLU 任务上至今仍具竞争力——对于某些强调高吞吐、低延迟的场景，BERT 比 LLM 更具实用性。

## 30.3 GPT

![GPT 模型架构](/images/ch30_5_2_1.png)
：自回归生成的先驱

**GPT**（Generative Pre-trained Transformer）由 OpenAI 于 2018 年发布，是第一个明确采用"预训练 + 微调"范式的 Decoder-Only 模型。虽然早期性能略逊于 BERT，但 OpenAI 坚定地沿着"扩大规模"的路线走下去，最终通过 GPT-3 和 ChatGPT 开启了大模型时代。

### 30.3.1 架构：Decoder-Only

GPT 只使用 Transformer 的 **Decoder** 部分，但移除了 Encoder-Decoder 交叉注意力层（因为没有 Encoder 输入），只保留**掩码自注意力**和**前馈网络**两个子层。

关键结构特点：

- **掩码自注意力**：通过因果掩码（Causal Mask）确保每个 token 只能看到它左边的 token，不能"偷看"未来信息。
- **Pre-Norm**（GPT-2/3）：将 LayerNorm 放在子层之前而非之后（`x + SubLayer(LayerNorm(x))`），缓解深层网络的梯度消失问题。
- **可学习的位置嵌入**：使用可学习的绝对位置编码。

### 30.3.2 预训练任务：CLM（因果语言模型）

GPT 的预训练任务是**因果语言模型（Causal Language Model, CLM）**——预测下一个 token：

$$P(x_1, ..., x_T) = \prod_{t=1}^{T} P(x_t | x_1, ..., x_{t-1})$$

这个看似简单的目标，迫使模型学习语法结构、事实知识、上下文逻辑等深层语言规律。CLM 的优势在于：

- **与生成任务天然匹配**：预训练目标和下游生成任务完全一致，没有 BERT 的预训练-微调不一致问题。
- **数据无限**：任何文本都可以直接用于 CLM 训练，无需任何标注或特殊处理。

### 30.3.3 从 GPT-1 到 GPT-3：规模的力量

| 模型 | Decoder 层数 | 隐藏维度 | 参数量 | 预训练数据 | 上下文长度 |
|------|------------|---------|--------|-----------|-----------|
| GPT-1 | 12 | 768 | 0.12B | 5GB | 512 |
| GPT-2 | 48 | 1600 | 1.5B | 40GB | 1024 |
| GPT-3 | 96 | 12288 | 175B | 570GB | 2048 |

**GPT-1（2018）**：开创性地提出 Decoder-Only 预训练模型，但规模和 BERT-Base 相当，性能略逊。

**GPT-2（2019）**：将规模扩大了一个数量级，首次提出**零样本学习（Zero-shot）**——不微调模型，直接通过描述任务让模型完成。虽然效果有限，但展示了模型规模增长带来的潜力。

**GPT-3（2020）**：将参数量推到 1750 亿，展现了强大的**少样本学习（Few-shot）**和**上下文学习（In-context Learning）**能力。通过在提示中提供 3-5 个示例，GPT-3 无需微调就能在各种任务上取得接近甚至超过微调模型的效果。这标志着 NLP 范式从"预训练 + 微调"向"预训练 + 提示"的转变。

## 30.4 T5

![T5 模型架构](/images/ch30_5_3_1.png)
：万物皆文本的统一框架

**T5**（Text-to-Text Transfer Transformer）由 Google 于 2020 年提出，回归了 Transformer 原始的 **Encoder-Decoder** 架构。它的核心理念是将所有 NLP 任务统一为"文本到文本"的转换问题。

### 30.4.1 Text-to-Text 统一框架

在 T5 之前，不同任务需要不同的模型结构和输出层。T5 打破了这一模式——无论什么任务，输入是文本，输出也是文本：

- **翻译**：输入 `"translate English to German: The house is wonderful."` → 输出 `"Das Haus ist wunderbar."`
- **情感分类**：输入 `"sentiment: This movie is great."` → 输出 `"positive"`
- **摘要**：输入 `"summarize: ..." ` → 输出摘要文本
- **回归任务**：输入 `"stsb sentence1: ... sentence2: ..."` → 输出 `"4.5"`（直接生成数字文本）

这种设计的巧妙之处在于：通过**任务前缀（Task Prefix）**区分不同任务，同一个模型可以处理几乎所有 NLP 问题。这其实就是大语言模型时代**提示词**的雏形——T5 证明了我们可以通过改变输入文本来"编程"模型，而不仅仅是改变模型结构。

### 30.4.2 预训练任务：Span Corruption

T5 设计了独特的预训练目标——**Span Corruption（片段破坏与重构）**：

1. **破坏**：在输入文本中随机选中连续的片段，替换为哨兵符（如 `<extra_id_0>`、`<extra_id_1>`）。
2. **重构**：让 Decoder 生成被遮盖的片段。

示例：

- **原始文本**：`黑神话悟空是一款以中国神话为背景的动作角色扮演游戏。`
- **Encoder 输入**：`黑神话悟空是一款<extra_id_0>的动作<extra_id_1>游戏。`
- **Decoder 输出**：`<extra_id_0>以中国神话为背景<extra_id_1>角色扮演<extra_id_2>`

与 BERT 的单字掩码相比，Span Corruption 遮盖的是连续片段（平均长度 3），迫使模型理解更长的上下文依赖。

### 30.4.3 关键技术改进

T5 在微观层面引入了多项优化，其中许多成为了后续大模型的标配：

- **相对位置编码**：采用分桶（Bucketing）策略——近距离精确区分，远距离通过对数映射归入同一"桶"。位置编码作为偏置直接加在注意力分数上，而非嵌入输入中。
- **简化版 LayerNorm**：去除加性偏置，仅保留缩放操作。
- **SentencePiece 分词器**：直接在原始文本上训练，语言无关，天然支持多语言。

## 30.5 三种范式对比

| 特性 | BERT (Encoder-Only) | GPT (Decoder-Only) | T5 (Encoder-Decoder) |
|------|--------------------|--------------------|---------------------|
| **注意力类型** | 双向自注意力 | 因果掩码自注意力 | Encoder 双向 + Decoder 因果 |
| **预训练任务** | MLM + NSP | CLM（预测下一个 token） | Span Corruption |
| **擅长任务** | 文本分类、NER、抽取式问答 | 文本生成、对话、续写 | 翻译、摘要、多任务统一 |
| **应用范式** | 预训练 + 微调 | 提示（Zero/Few-shot） | 任务前缀 + 微调 |
| **代表模型** | BERT、RoBERTa、ALBERT | GPT 系列、LLaMA | T5、mT5、Flan-T5 |
| **生成能力** | ❌ 不支持原生生成 | ✅ 天然擅长生成 | ✅ 支持生成 |

从历史演进看，**Decoder-Only** 架构最终胜出，成为当前大语言模型（LLM）的标准架构。原因在于：

1. CLM 预训练目标与生成任务完全一致，不存在预训练-微调不一致的问题。
2. 当模型规模足够大时，CLM 训练的模型在理解任务上也能超越 MLM 模型。
3. Decoder-Only 结构更简洁，更易于规模化训练。

## 30.6 HuggingFace 生态

![HuggingFace 生态](/images/ch30_5_4_1.png)


**Hugging Face** 已从一家 NLP 初创公司演变为 AI 领域的基础设施。它构建了涵盖模型开发全生命周期的完整生态系统。

### 30.6.1 核心库

- **Transformers**：生态系统的引擎，提供统一的 API 来下载、加载和使用预训练模型。支持 BERT、GPT、T5 等几乎所有主流架构。
- **Tokenizers**：连接文本与模型的桥梁，底层由 Rust 编写，提供极致的分词速度（Fast Tokenizers）。
- **Datasets**：数据处理的加速器，基于 Apache Arrow 实现内存映射，即使 TB 级数据集也能高效处理。
- **Evaluate**：标准化的模型评估框架，提供 Accuracy、F1、BLEU、ROUGE 等常用指标。

### 30.6.2 Hugging Face Hub

Hub 是 AI 领域的"GitHub"，集成了：

- **Models**：托管数十万个预训练模型，涵盖 NLP、CV、Audio 等多模态领域。
- **Datasets**：托管各类公开数据集，支持预览和流式传输。
- **Spaces**：使用 Gradio 或 Streamlit 快速构建模型演示 Web 应用。

### 30.6.3 核心 API 模式

使用 Transformers 库的标准流程遵循三个步骤：

1. **Tokenizer**：文本 → Token IDs（数字序列）
2. **Model**：Token IDs → Logits（模型输出）
3. **Post-processing**：Logits → 概率/标签

`AutoClass` 是库设计的精髓——`AutoTokenizer.from_pretrained("bert-base-chinese")` 会自动推断正确的分词器和模型架构。`from_pretrained` 既支持从 Hub 在线下载，也支持从本地加载。

### 30.6.4 Trainer 训练框架

对于模型微调，Transformers 提供了高度封装的 **Trainer** API，集成了混合精度训练、梯度累积、分布式训练等工程化特性。使用 Trainer 的标准流程是"三步走"：

1. **准备组件**：实例化 Model、Dataset 和 Tokenizer。
2. **配置参数**：通过 `TrainingArguments` 定义超参数（Batch Size、学习率、Epoch、保存策略等）。
3. **启动训练**：实例化 `Trainer` 并调用 `train()` 即可。

Trainer 底层自动调用 **Accelerate** 库，同一套代码可以无缝运行在 CPU、单卡 GPU、多卡 GPU 甚至 TPU 上，无需手动处理分布式逻辑。对于追求更高灵活性的开发者，也可以手写 PyTorch 训练循环，但 Trainer 在大多数场景下已经足够好用。

### 30.6.5 Datasets 与 Evaluate

**Datasets** 库基于 Apache Arrow 格式实现内存映射，支持不将整个数据集加载到 RAM 就能高效处理。即使是 TB 级数据集也能在普通笔记本上运行。其核心功能是 `map` 方法，可以将预处理函数（如 Tokenizer）高效地应用到每个样本上，支持批处理和多进程并行。

**Evaluate** 库提供了标准化的评估接口，内置 Accuracy、F1、BLEU、ROUGE 等数十种常用指标，与 Trainer 无缝集成——只需定义 `compute_metrics` 函数传入 Trainer 即可。

## 30.7 实战：用 HuggingFace 跑一个 BERT 文本分类

下面的完整代码展示了如何使用 HuggingFace Transformers 加载预训练 BERT 模型并进行微调，完成文本分类任务。

```python
import torch
from transformers import AutoTokenizer, AutoModelForSequenceClassification
from transformers import TrainingArguments, Trainer
from datasets import load_dataset
import evaluate
import numpy as np

# ============================================================
# 1. 加载数据集（使用英文情感分类数据集演示）
# ============================================================
dataset = load_dataset("rotten_tomatoes")
print("数据集结构:", dataset)

# ============================================================
# 2. 加载分词器和模型
# ============================================================
checkpoint = "bert-base-uncased"
tokenizer = AutoTokenizer.from_pretrained(checkpoint)
model = AutoModelForSequenceClassification.from_pretrained(
    checkpoint, num_labels=2
)

# ============================================================
# 3. 数据预处理：文本 → Token IDs
# ============================================================
def tokenize_function(examples):
    return tokenizer(
        examples["text"],
        padding="max_length",
        truncation=True,
        max_length=128
    )

tokenized_datasets = dataset.map(tokenize_function, batched=True)

# 取小子集加速演示
small_train = tokenized_datasets["train"].shuffle(seed=42).select(range(200))
small_eval = tokenized_datasets["test"].shuffle(seed=42).select(range(100))

# ============================================================
# 4. 定义评估指标
# ============================================================
metric = evaluate.load("accuracy")

def compute_metrics(eval_pred):
    logits, labels = eval_pred
    predictions = np.argmax(logits, axis=-1)
    return metric.compute(predictions=predictions, references=labels)

# ============================================================
# 5. 配置训练参数
# ============================================================
training_args = TrainingArguments(
    output_dir="bert-classifier-output",
    eval_strategy="epoch",
    num_train_epochs=2,
    per_device_train_batch_size=16,
    per_device_eval_batch_size=16,
    learning_rate=2e-5,
    weight_decay=0.01,
    logging_steps=10,
    save_strategy="no",
)

# ============================================================
# 6. 创建 Trainer 并训练
# ============================================================
trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=small_train,
    eval_dataset=small_eval,
    compute_metrics=compute_metrics,
)

trainer.train()

# ============================================================
# 7. 评估模型
# ============================================================
results = trainer.evaluate()
print("评估结果:", results)

# ============================================================
# 8. 推理示例
# ============================================================
classifier = pipeline(
    "sentiment-analysis",
    model=model,
    tokenizer=tokenizer
)

test_texts = [
    "This movie is absolutely wonderful!",
    "Terrible film, waste of time.",
    "A decent movie with some good moments."
]

for text in test_texts:
    result = classifier(text)
    print("文本: {}  ->  预测: {}".format(text, result[0]["label"]))
```

运行这段代码，你将看到模型在情感分类任务上的训练过程和评估结果。整个流程展示了 HuggingFace 生态的核心优势：几行代码就能完成从数据加载、模型训练到推理的完整流程。

## 30.8 常见误区

**误区 1：BERT 比 GPT 更好（或反之）**

没有绝对的优劣，只有适用场景的不同。BERT 擅长理解（分类、实体识别），GPT 擅长生成（写作、对话）。在大模型时代，Decoder-Only 架构通过规模优势在理解任务上也超越了 BERT，但对于数据充足、强调高吞吐的特定任务，BERT 仍然更实用。

**误区 2：预训练就是"在大数据上训练"**

预训练的核心不仅是数据量大，更在于**无监督学习**（不需要人工标注）和**通用表示学习**（学到的知识可以迁移到多种任务）。仅仅在大数据上训练一个任务特定的模型不叫预训练。

**误区 3：微调一定需要大量数据**

得益于预训练阶段学到的丰富知识，微调通常只需要几百到几千条标注数据。这正是预训练范式的核心价值——通过数据杠杆降低下游任务的数据需求。

**误区 4：HuggingFace 只是一个模型下载网站**

HuggingFace 是一个完整的 AI 开发生态系统，涵盖模型、数据集、训练框架、评估工具、部署方案和社区协作平台。

---

## 30.9 小结

本章介绍了预训练语言模型的三大范式：

- **BERT（Encoder-Only）**：通过 MLM 学习双向语义，在 NLU 任务上统治了多年。
- **GPT（Decoder-Only）**：通过 CLM 学习自回归生成，从 GPT-1 到 GPT-3 展示了规模的力量，最终开创了 LLM 时代。
- **T5（Encoder-Decoder）**：将所有任务统一为文本到文本，是多任务学习的典范。

我们还介绍了 HuggingFace 生态，它将这些模型的能力以极低的门槛开放给所有开发者。

在下一章，我们将深入探讨大语言模型（LLM）的核心概念：规模定律、涌现能力和三阶段训练流程。
