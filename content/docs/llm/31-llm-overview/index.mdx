# 31 大模型概览：规模定律、涌现能力与训练三阶段

2022 年底，ChatGPT 的发布引爆了全球对大语言模型（Large Language Model, LLM）的关注。从 GPT-3 的 1750 亿参数到如今万亿参数级别的模型，LLM 展现出了远超传统预训练模型的能力——它们不仅能写文章、编代码，还能进行复杂的逻辑推理。

本章将回答三个核心问题：什么是大语言模型？为什么规模能带来质变？一个 LLM 是如何训练出来的？

---

## 31.1 什么是大语言模型

大语言模型（LLM）是一种在**海量无监督文本**上预训练的、拥有**数十亿甚至数千亿参数**的语言模型。它与传统预训练模型（如 BERT、T5）使用相似的架构（主要是 Decoder-Only），但在三个维度上实现了量级的跨越：

| 维度 | 传统 PLM（如 BERT） | LLM（如 GPT-3） |
|------|-------------------|-----------------|
| **参数量** | 1 亿 ~ 3 亿 | 数十亿 ~ 数千亿 |
| **训练数据** | 数十 GB（数十亿 token） | 数百 GB ~ 数 TB（数千亿 token） |
| **算力** | 数十块 GPU，数天 | 数千块 GPU，数周到数月 |

参数量、数据量和算力是 LLM 的三要素，三者缺一不可。更大的参数赋予模型更强的表达能力，更多的数据让模型学到更丰富的知识，而充足的算力则是前两者的保障。

一般认为，GPT-3（1750 亿参数）是 LLM 的开端。但"大"的边界在不断变化——如今即使是 70 亿参数的模型（如 LLaMA-7B），在经过精心训练后也能展现出令人印象深刻的能力。广义上，只要模型展现出**涌现能力**，即在复杂任务上表现出远超传统预训练模型的能力，都可以称之为 LLM。

## 31.2 规模定律

![规模定律](/images/ch31_2-0.jpg)
（Scaling Laws）

为什么"大力出奇迹"？答案藏在**规模定律**中。

### 31.2.1 Kaplan et al. 2020 的发现

2020 年，OpenAI 的 Kaplan 等人发现了语言模型性能与三个因素之间的**幂律关系**。具体来说，模型的交叉熵损失 $L$ 与参数量 $N$、数据量 $D$、计算量 $C$ 之间存在平滑的幂律关系：

$$L(N) \approx \left(\frac{N_c}{N}\right)^{\alpha_N}$$

$$L(D) \approx \left(\frac{D_c}{D}\right)^{\alpha_D}$$

$$L(C) \approx \left(\frac{C_c}{C}\right)^{\alpha_C}$$

其中 $N_c, D_c, C_c, \alpha_N, \alpha_D, \alpha_C$ 是正常数。这意味着：

- 模型性能（以损失衡量）会随着参数量、数据量或计算量的增加而**持续改善**，且改善遵循可预测的幂律曲线。
- **没有明显的天花板**：在实验范围内，增大模型规模总能带来性能提升。

更重要的是，Kaplan 等人发现**模型大小比数据量更重要**——建议在固定计算预算下，优先增大模型参数量。

### 31.2.2 Chinchilla 最优

2022 年，DeepMind 的 Hoffmann 等人通过更大规模的实验修正了 Kaplan 的结论。他们的 **Chinchilla** 论文指出：

- Kaplan 的结论过于偏向增大模型参数，实际训练中数据量同样重要。
- 最优的训练策略应该是**模型参数和训练 token 数同步增长**。
- 具体而言，最优的训练 token 数约为模型参数的 **20 倍**。

这意味着：

- 一个 10B 参数的模型，应该用约 200B token 来训练。
- 一个 70B 参数的模型，应该用约 1.4T token 来训练。

Chinchilla 的 70B 模型用 1.4T token 训练，在多项评测上超越了 Gopher 的 280B 模型（只用了 300B token）——用不到一半的参数量和更低的推理成本，取得了更好的效果。

这个发现深刻影响了后续 LLM 的训练策略。LLaMA 系列模型就采用了 Chinchilla 的思路，使用远超传统比例的训练 token 数，从而在较小的参数量下实现了优异的性能。

### 31.2.3 Scaling Laws 的实践意义

规模定律的意义不仅在于"告诉我们大模型更好"，更在于它提供了**可预测性**：

- 可以通过小规模实验的结果，**外推预测**大规模训练的性能。
- 可以在给定的计算预算下，**优化分配**参数量、数据量和训练时长。
- 减少了大模型训练的试错成本——不需要真的训练一个 175B 的模型来猜测效果。

## 31.3 涌现能力

![涌现能力示例](/images/ch31_2-1.jpg)
（Emergent Abilities）

规模定律告诉我们，增大模型可以持续改善性能。但真正让人兴奋的是：当模型规模达到某个临界点时，会突然展现出**小模型完全不具备的能力**——这就是**涌现能力**。

### 31.3.1 什么是涌现

涌现能力的定义是：**在小模型中不明显或不存在，但在大模型中突然出现的能力**。可以类比物理学中的相变——水在 99°C 时仍然是液态，但到达 100°C 时突然沸腾。模型的某些能力也有类似的"临界点"。

这种现象的出现意味着：模型不仅仅是"量变"（更大的模型在同样的任务上表现更好），更是"质变"（更大的模型能做到小模型完全做不到的事情）。

### 31.3.2 典型涌现能力

**思维链（Chain-of-Thought, CoT）推理**：

当模型规模超过约 100B 参数时，通过在提示中加入推理步骤示例，模型能够展示出逐步推理的能力。例如：

```
问：Roger 有 5 个网球，他又买了 2 罐网球，每罐 3 个。他现在有多少个网球？
答：Roger 开始有 5 个球。2 罐网球，每罐 3 个，共 6 个。5 + 6 = 11。答案是 11。
```

小模型在给出这种"思维链示例"后，反而会表现更差（因为它们无法理解中间推理步骤），而大模型则能准确地按照推理步骤得出正确答案。

**少样本学习（Few-shot Learning）**：

GPT-3 展示的强大少样本能力——通过在提示中给出 3-5 个示例，模型就能在未见过的任务上表现良好——就是一种涌现能力。小模型即使给出示例也无法有效学习，但大模型可以"从上下文中学习"。

**指令遵循（Instruction Following）**：

经过指令微调的大模型能够理解并遵循自然语言描述的指令，即使指令涉及模型从未见过的任务类型。这种泛化能力在小模型中几乎不存在。

### 31.3.3 涌现的争议

涌现能力的概念并非没有争议。2023 年的研究（Schaeffer et al.）提出了一个重要质疑：**涌现能力可能只是评估指标的假象**。

具体来说，当使用**非线性指标**（如精确匹配 Accuracy）评估时，模型性能呈现"突然跃升"的涌现模式；但如果换用**连续指标**（如 Token-level Accuracy 或 Brier Score），性能提升实际上是**平滑且连续的**。

换句话说，模型可能从很早开始就在逐步学习这些能力，只是因为评估指标的粗糙性（一个 token 错了，整个答案就判错），才表现为"突然出现"。

这个争议提醒我们：对涌现能力的理解可能比表面上看到的更复杂。但无论如何，大模型在复杂任务上的显著优势是实实在在的。

## 31.4 LLM 训练三阶段

![LLM 训练三阶段](/images/ch31_2-3.png)


训练一个完整的 LLM 需要经过三个阶段：**预训练（Pretrain）**、**有监督微调（SFT）**和**偏好对齐（RLHF/DPO）**。

```
阶段一：预训练          阶段二：SFT           阶段三：RLHF/DPO
┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│  海量无监督   │      │  指令-回复    │      │  人类偏好     │
│  文本语料     │ ──→  │  有监督数据   │ ──→  │  反馈数据     │
│             │      │             │      │             │
│  学习语言    │      │  学习指令    │      │  学习价值观   │
│  知识和规律  │      │  遵循能力    │      │  对齐        │
└─────────────┘      └─────────────┘      └─────────────┘
    输出：基座模型        输出：指令模型         输出：对齐模型
   （只会续写）        （能理解指令）        （安全、有用、无害）
```

### 31.4.1 阶段一：预训练（Pretrain）

预训练是 LLM 训练中最核心、工程量最大的阶段。它的本质是用海量无监督文本训练模型学习"预测下一个 token"：

$$\mathcal{L}_{\text{pretrain}} = -\sum_{t=1}^{T} \log P(x_t | x_1, ..., x_{t-1}; \theta)$$

**关键要素**：

- **模型架构**：几乎所有主流 LLM 都采用 Decoder-Only 的类 LLaMA 架构，使用 RMSNorm、SwiGLU 激活函数、RoPE 位置编码等优化技术。
- **预训练任务**：因果语言模型（CLM），即预测下一个 token。
- **训练数据**：数百 GB 到数 TB 的混合语料，包括 CommonCrawl、C4、GitHub、Wikipedia、书籍等。数据配比是各厂商的"核心秘籍"。
- **计算资源**：百亿级模型通常需要 1024 张 A100 训练一个月以上。分布式训练框架（如 DeepSpeed、Megatron-LM）是必需品。

预训练赋予了 LLM 海量的知识和强大的语言能力，但它只是一个"博览群书却不求甚解的书生"——它能流畅地续写任何文本，却不知道如何理解和遵循用户的指令。

#### 分布式训练

由于模型规模巨大，单张 GPU 无法容纳完整的 LLM 参数，分布式训练框架是必需品。主流的并行策略包括：

- **数据并行**：每张 GPU 存放完整模型副本，处理不同的数据批次。适合模型能放进单卡的场景。
- **模型并行**：将模型的不同层分配到不同 GPU 上。适合模型太大的场景。
- **张量并行**：将单个层的参数矩阵拆分到多张 GPU 上，进一步细粒度并行。
- **ZeRO（零冗余优化器）**：DeepSpeed 的核心策略，通过分片存储模型参数、梯度和优化器状态来减少显存占用。ZeRO-1 只分片优化器状态，ZeRO-2 还分片梯度，ZeRO-3 连模型参数也分片。

主流分布式训练框架包括 DeepSpeed（使用最广）、Megatron-LM（NVIDIA）、ColossalAI 等。

#### 数据质量与清洗

预训练数据的质量往往比体量更重要。数据处理流程通常包括：

1. **文档准备**：从网页 HTML 提取纯文本，过滤有害 URL，识别语种。
2. **语料过滤**：去除低质量、无意义、有毒有害的内容（乱码、广告等）。可以通过基于模型的分类器或基于启发式的规则。
3. **语料去重**：大量重复文本会显著影响模型泛化能力。基于 hash 算法或子串匹配进行去重。

实验证明，高质量的 627B SlimPajama 数据集训练出的模型，效果优于 1T 的 RedPajama 数据集。

### 31.4.2 阶段二：有监督微调（SFT）

SFT 的目标是教会 LLM **指令遵循能力**——让模型能够理解用户的指令并给出有意义的回复。

训练数据是**指令-回复对**：

```json
{
    "instruction": "将下列文本翻译成英文：",
    "input": "今天天气真好",
    "output": "Today is a nice day!"
}
```

SFT 的核心要点：

- **数据质量和多样性**比数据量更重要。InstructGPT 使用了 10 种类型的指令数据，覆盖文本生成（45.6%）、问答（12.4%）、头脑风暴（11.2%）等。
- **多轮对话能力**完全来自 SFT 阶段。通过将训练数据构造成多轮对话格式，模型学会利用历史信息生成回复。
- **格式很重要**：SFT 通常使用特定的模板格式（如 `### Instruction:\n{content}\n\n### Response:\n`），让模型学会区分指令和回复的边界。

经过 SFT 的模型从"只会续写"变成了"能理解并遵循指令"——这就是从 GPT-3 到 ChatGPT 的关键一步。

#### 多轮对话的构造

SFT 还赋予了模型**多轮对话**能力——在每次回复时参考之前的对话历史。训练数据需要构造成多轮对话格式。最合理的方式是要求模型预测每一轮的输出，而不仅仅是最后一轮：

```
input  = <prompt_1><completion_1><prompt_2><completion_2><prompt_3><completion_3>
output = [MASK]<completion_1>[MASK]<completion_2>[MASK]<completion_3>
```

这之所以可行，是因为 LLM 本质仍是 CLM 任务，单向注意力计算时前轮的输出预测不会影响后轮的预测。目前几乎所有 LLM 都使用这种多轮对话格式进行 SFT。

#### SFT 的数据挑战

高质量指令数据的获取是 SFT 的核心挑战。不同于预训练使用的无监督语料，SFT 需要有监督的指令-回复对。人工标注成本极高，因此社区探索了多种替代方案：

- **Self-Instruct**：用强模型（如 GPT-4）生成指令数据。经典数据集 Alpaca 就是通过 ChatGPT 生成的 52K 条指令数据。
- **开源指令数据集**：如 ShareGPT（用户分享的真实对话）、OpenAssistant 等。
- **数据质量优先**：研究表明，少量高质量数据（如 LIMA 的 1000 条精选数据）可能比大量低质量数据更有效。

### 31.4.3 阶段三：RLHF / DPO

SFT 让模型学会了遵循指令，但模型的回复可能仍然不够安全、不够有用、甚至可能存在偏见。**RLHF**（Reinforcement Learning from Human Feedback）通过人类反馈进一步优化模型，使其输出符合人类的价值观偏好——即**安全、有用、无害**。

RLHF 分为两个步骤：

**步骤一：训练奖励模型（Reward Model, RM）**

RM 是一个学习人类偏好的评分器。训练数据是人工标注的**偏好对比**：

```json
{
    "prompt": "如何提高编程能力？",
    "chosen": "多写代码、阅读优秀源码、参与开源项目...",
    "rejected": "编程能力不重要，用 AI 写就行了。"
}
```

RM 学会对模型的回复打分，分数越高表示越符合人类偏好。

**步骤二：PPO 强化学习训练**

使用 PPO（Proximal Policy Optimization）算法，以 RM 的打分为奖励信号，优化 LLM 的生成策略。这个过程中同时存在四个模型：Actor Model（被训练的 LLM）、Ref Model（不更新的参考 LLM，用于防止偏离太远）、Reward Model（打分）和 Critic Model（评估价值函数）。

RLHF 的资源消耗巨大（四个模型同时在 GPU 上运行），且训练不稳定。因此，**DPO**（Direct Preference Optimization）作为轻量替代方案被提出。DPO 将偏好学习转化为监督学习问题，直接在偏好数据上优化模型，无需训练 RM 和进行强化学习，只需要两个模型即可完成。

## 31.5 主流 LLM 家族

### GPT 系列（OpenAI）

- **GPT-3.5/4**：闭源模型，ChatGPT 的基座。GPT-4 支持多模态（文本 + 图像），在多项评测中达到人类水平。
- **GPT-4o**：支持文本、图像、音频的多模态输入输出，推理速度大幅提升。
- **o1/o3 系列**：专注推理能力的模型，通过"思考"过程解决复杂问题。
- 特点：持续引领行业，性能最强，但完全闭源。

### LLaMA 系列（Meta）

- **LLaMA-1（2023.02）**：7B/13B/30B/65B，在 1T token 上训练，开源引爆社区。
- **LLaMA-2（2023.07）**：7B/13B/70B，在 2T token 上训练，引入 GQA，上下文扩展到 4096。
- **LLaMA-3（2024.04）**：8B/70B，在 15T token 上训练，词表扩展到 128K。
- **LLaMA-3.1（2024.07）**：8B/70B/405B，405B 为开源最大稠密模型，支持 128K 上下文。
- 特点：开源，成为社区最重要的基座模型，大量衍生模型（Alpaca、Vicuna 等）。

### Qwen 系列（阿里）

- **Qwen-1.5/2/2.5**：覆盖 0.5B 到 72B 多种规模，中英双语能力突出。
- **Qwen-2.5-Coder**：专注代码生成，在代码评测中表现优异。
- **Qwen-2.5-Math**：专注数学推理，在数学基准上达到领先水平。
- 特点：中文能力强，开源，支持工具调用和代码生成。

### DeepSeek 系列

- **DeepSeek-V2/V3**：采用 MoE（Mixture of Experts）架构，以更低的训练成本实现优异性能。
- **DeepSeek-R1**：在推理任务上表现出色。
- **DeepSeek-Coder**：专注代码生成的系列模型，在编程任务上表现突出。
- 特点：技术创新突出（MoE、MLA 等），性价比高。

### Mistral / Mixtral（Mistral AI）

- **Mistral-7B**：小参数量大能力，多项评测超越 LLaMA-2-13B。
- **Mixtral-8x7B**：采用 MoE 架构，8 个专家中每次激活 2 个。
- **Mistral-Large**：123B 参数的稠密模型，对标 GPT-4。
- 特点：欧洲团队，注重效率和开源。

### 其他值得关注的模型

- **Claude 系列（Anthropic）**：闭源，以安全性和长文本处理著称，Claude-3.5 Sonnet 在多项评测中表现优异。
- **Gemini 系列（Google）**：闭源，原生多模态（文本、图像、音频、视频），Gemini Ultra 在 MMLU 基准上首次超越人类专家。
- **Yi 系列（零一万物）**：开源，中英双语能力强，支持 200K 超长上下文。
- **GLM 系列（智谱）**：开源，国内最早的大模型之一，从 GLM 预训练任务发展到标准 CLM 架构。
- **InternLM（上海 AI Lab）**：开源，支持超长上下文（200K），在中文任务上表现优异。

### 31.6.1 多语言与长上下文

LLM 由于使用海量多语言语料预训练，天然具备多语言能力，但在不同语言上的表现差异较大。以 GPT-4 为代表的英文主导模型在英文上显著优于中文，而国内模型（如 Qwen、DeepSeek）则针对中文进行了额外优化。

长上下文处理是 LLM 的另一重要能力。传统 PLM（如 BERT、T5）的最大上下文长度通常为 512 token，而 LLM 通过 RoPE（旋转位置编码）等技术，将上下文扩展到 4K、8K 甚至 128K 以上。长上下文能力使 LLM 能够处理长文档阅读、多轮对话等复杂场景。

### 31.6.2 多模态扩展

通过为 LLM 增加视觉编码器和适配器层，并在图文数据上进行微调，可以打造支持文本 + 图像双模态的模型。GPT-4V、Gemini、Qwen-VL 等多模态大模型已经展示了强大的图文理解和生成能力。

### 31.6.3 幻觉问题

**幻觉（Hallucination）**是 LLM 的固有缺陷——模型可能生成看似合理但实际虚假的信息。例如在生成学术论文引用时，LLM 经常捏造不存在的论文和作者。幻觉在医学、金融等强调准确性的领域可能造成严重后果。目前的缓解方法包括 RAG（检索增强生成）、提示约束等，但无法彻底根除。

## 31.7 开源 vs 闭源

| 维度 | 开源 LLM | 闭源 LLM |
|------|---------|---------|
| **代表** | LLaMA、Qwen、DeepSeek | GPT-4、Claude、Gemini |
| **优势** | 可本地部署、可定制微调、社区生态活跃 | 性能通常最强、无需运维、开箱即用 |
| **劣势** | 顶级性能仍有差距、需要算力部署 | 数据隐私风险、无法深度定制、成本持续 |
| **适用场景** | 需要定制化、数据敏感、离线场景 | 快速原型、通用任务、无 GPU 资源 |

开源与闭源的竞争推动了整个领域的快速发展。开源模型通过社区的持续改进，在很多任务上已经接近甚至达到闭源模型的水平，同时为研究者和开发者提供了透明、可控的技术基础。

## 31.8 常见误区

**误区 1：参数越多，模型一定越好**

Chinchilla 定律表明，参数量和训练数据量需要**匹配增长**。一个用 300B token 训练的 280B 模型，可能不如用 1.4T token 训练的 70B 模型。数据质量和训练策略同样关键。

**误区 2：涌现能力是"魔法"**

涌现能力的"突然出现"可能部分源于评估指标的粗糙性。使用连续指标评估时，性能提升往往是平滑的。但大模型在复杂任务上的显著优势仍然是真实的。

**误区 3：RLHF 是让模型"学会思考"**

RLHF 的核心作用是**对齐**——让模型的输出更符合人类的价值观和偏好，而不是提升模型的基础能力。模型的基础知识和能力主要来自预训练阶段。

**误区 4：三阶段训练是严格顺序的**

虽然三阶段通常按顺序进行，但实践中会有迭代。例如，可以在 RLHF 后继续进行 SFT（称为"拒绝采样"），或者在多个阶段之间交替训练。

## 31.9 小结

本章介绍了大语言模型的核心概念：

- **LLM 的定义**：数十亿参数、数 T token 训练数据、展现涌现能力的语言模型。
- **规模定律**：模型性能与参数量、数据量、算力之间存在可预测的幂律关系。Chinchilla 最优表明参数和数据应同步增长。
- **涌现能力**：当模型规模超过临界点时，会突然展现出小模型不具备的能力（思维链推理、少样本学习、指令遵循）。但"突然出现"可能是评估指标的假象。
- **训练三阶段**：预训练（学习知识）→ SFT（学习指令遵循）→ RLHF/DPO（学习人类偏好）。每一步都有其不可替代的作用。
- **主流模型家族**：GPT、LLaMA、Qwen、DeepSeek、Mistral 各有特色，开源与闭源竞争推动领域快速发展。

在后续章节中，我们将从零开始实践 LLM 的完整训练流程，包括预训练、SFT 和 RLHF。
