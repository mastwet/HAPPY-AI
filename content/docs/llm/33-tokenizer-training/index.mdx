# 33 训练 Tokenizer

Tokenizer 是 LLM 的"眼睛"——模型看到的不是文字，而是 Token 序列。一个好的 Tokenizer 直接决定模型的性能上限。

## 为什么 Tokenizer 很重要

很多人忽略了 Tokenizer，但它影响深远：

- **Token 数量**：同一段文本，不同 Tokenizer 产出的 Token 数不同 → 影响训练成本和推理速度
- **语义粒度**：太粗（整词）导致 OOV，太细（字符）导致序列过长
- **多语言能力**：中文在英文主导的 Tokenizer 中可能被拆成很多碎片

GPT-2 的 BPE Tokenizer 对中文特别不友好（一个汉字可能被拆成 2-3 个 Token），而 Qwen 的 Tokenizer 对中文做了专门优化。

## Tokenizer 的三种类型

| 类型 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| Word-based | 按空格/词典切分 | 语义完整 | 词表巨大、OOV 严重 |
| Char-based | 按字符切分 | 词表小、无 OOV | 序列过长、语义弱 |
| Subword-based | 按子词切分 | 词表适中、无 OOV | 需要训练 |

**Subword** 是当前 LLM 的标准选择。三种主流算法：BPE（GPT 系列）、WordPiece（BERT）、SentencePiece（LLaMA）。

## BPE 算法详解

BPE（Byte Pair Encoding，字节对编码）是最常用的 Subword 算法。

### 核心思想

反复合并出现频率最高的相邻字符对，直到达到目标词表大小。

### 算法步骤

```text
输入：训练语料、目标词表大小 V
1. 初始化词表 = 所有单个字符（或字节）
2. 统计所有相邻 Token 对的出现频率
3. 找到频率最高的 Token 对，合并为新 Token
4. 将新 Token 加入词表
5. 重复 2-4，直到词表大小达到 V
```

### Python 实现

```python
from collections import Counter, defaultdict


def get_stats(vocab):
    """统计所有相邻 Token 对的频率"""
    pairs = defaultdict(int)
    for word, freq in vocab.items():
        symbols = word.split()
        for i in range(len(symbols) - 1):
            pairs[(symbols[i], symbols[i + 1])] += freq
    return pairs


def merge_vocab(pair, vocab):
    """合并频率最高的 Token 对"""
    new_vocab = {}
    bigram = " ".join(pair)
    replacement = "".join(pair)
    for word, freq in vocab.items():
        new_word = word.replace(bigram, replacement)
        new_vocab[new_word] = freq
    return new_vocab


def bpe(text, num_merges=50):
    """BPE 训练"""
    # 初始化：按字符拆分，统计词频
    words = text.lower().split()
    vocab = Counter()
    for w in words:
        # 每个字符用空格分隔，末尾加 </w> 标记词边界
        vocab[" ".join(list(w)) + " </w>"] += 1

    for i in range(num_merges):
        pairs = get_stats(vocab)
        if not pairs:
            break
        best = max(pairs, key=pairs.get)
        vocab = merge_vocab(best, vocab)
        print(f"Step {i+1}: merge {best} -> {''.join(best)}")

    return vocab


# 示例
text = "low lower lowest low low low newer newer newer wider wider"
result = bpe(text, num_merges=10)
```

### 分词（Tokenization）

训练完成后，对新文本分词：

```python
def tokenize(text, merges):
    """用学到的合并规则对新文本分词"""
    tokens = list(text.lower())
    for pair in merges:
        i = 0
        while i < len(tokens) - 1:
            if tokens[i] == pair[0] and tokens[i + 1] == pair[1]:
                tokens[i] = pair[0] + pair[1]
                del tokens[i + 1]
            else:
                i += 1
    return tokens
```

## WordPiece 与 SentencePiece

### WordPiece（BERT 使用）

与 BPE 类似，但合并标准不同：

- BPE：选频率最高的对
- WordPiece：选使语言模型似然增加最多的对

$$\text{score}(x, y) = \frac{\text{freq}(xy)}{\text{freq}(x) \times \text{freq}(y)}$$

WordPiece 用 `##` 前缀标记非词首子词：`playing → play + ##ing`

### SentencePiece（LLaMA、Qwen 使用）

- **语言无关**：直接在原始文本上训练，不依赖预分词
- **可逆**：Token 序列可以完美还原原始文本
- **BPE 和 Unigram** 两种模式

```python
import sentencepiece as spm

# 训练
spm.SentencePieceTrainer.train(
    input="corpus.txt",
    model_prefix="my_tokenizer",
    vocab_size=32000,
    model_type="bpe",
    character_coverage=0.9995,  # 中文建议高覆盖率
)

# 使用
sp = spm.SentencePieceProcessor(model_file="my_tokenizer.model")
tokens = sp.encode("大语言模型是人工智能的重要方向", out_type=str)
print(tokens)
# ['▁大', '语言', '模型', '是', '人工', '智能', '的', '重要', '方向']
```

## 用 HuggingFace Tokenizers 训练

HuggingFace Tokenizers 库提供了高效的 Rust 实现：

```python
from tokenizers import Tokenizer, models, trainers, pre_tokenizers

# 初始化 BPE Tokenizer
tokenizer = Tokenizer(models.BPE())
tokenizer.pre_tokenizer = pre_tokenizers.ByteLevel(add_prefix_space=False)

# 训练
trainer = trainers.BpeTrainer(
    vocab_size=32000,
    min_frequency=2,
    special_tokens=["[PAD]", "[UNK]", "[CLS]", "[SEP]", "[MASK]"],
)
tokenizer.train(files=["corpus.txt"], trainer=trainer)

# 保存和使用
tokenizer.save("my_tokenizer.json")
encoded = tokenizer.encode("Hello, world!")
print(encoded.tokens)
```

## 中文分词的特殊挑战

中文没有空格分隔，Tokenization 更困难：

| 挑战 | 说明 | 解决方案 |
|------|------|---------|
| 无空格 | "我喜欢苹果" 怎么切？ | SentencePiece 直接处理 |
| 字词粒度 | "机器学习" 是一个词还是两个？ | BPE 自动学习 |
| 多语言混合 | "用Python写一个LLM" | 字节级 BPE |

**最佳实践**：用 SentencePiece 训练，`character_coverage=0.9995`，确保中文字符不被丢弃。

## Tokenizer 评测

| 指标 | 含义 | 目标 |
|------|------|------|
| 压缩率 | 每个 Token 平均覆盖多少字符 | 越高越好 |
| OOV 率 | 无法表示的词占比 | 越低越好 |
| 多语言平衡 | 各语言 Token 数比例 | 越均衡越好 |

```python
# 测试压缩率
text = "大语言模型正在改变世界"
tokens = sp.encode(text, out_type=str)
compression = len(text) / len(tokens)
print(f"压缩率: {compression:.2f} chars/token")
```

## 常见误区

1. **认为 Tokenizer 不重要**：Token 质量直接决定模型"视力"
2. **词表越大越好**：太大的词表增加 Embedding 参数，低频 Token 训练不充分
3. **用英文 Tokenizer 处理中文**：一个汉字被拆成 2-3 个字节，效率极低
4. **忽略特殊 Token**：`[PAD]`、`[UNK]`、`[CLS]` 等必须正确处理
5. **训练数据太少**：Tokenizer 需要在大规模语料上训练才能学到好的切分

## 小结

| 概念 | 说明 |
|------|------|
| BPE | 字节对编码，频率最高的相邻对合并 |
| WordBERT | 似然最大的相邻对合并，`##` 标记非词首 |
| SentencePiece | 语言无关，可逆，LLaMA/Qwen 标配 |
| HuggingFace Tokenizers | Rust 实现，高效训练和推理 |
| 词表大小 | 32K-128K 是常见范围 |
