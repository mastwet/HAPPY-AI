# 36 PEFT / LoRA

全量微调一个 7B 模型需要 ~56GB 显存（FP16），加上优化器需要 100GB+。PEFT（参数高效微调）让普通开发者也能微调大模型——只需更新 0.1% 的参数。

## 全量微调的问题

| 问题 | 说明 |
|------|------|
| 显存不足 | 7B 模型 FP16 全量微调需 100GB+ |
| 存储开销 | 每个任务存一份完整模型（14GB） |
| 灾难性遗忘 | 全量更新容易丢失预训练知识 |
| 训练不稳定 | 参数多，优化难度大 |

## PEFT 技术总览

![PEFT 技术总览](/images/ch36_11_1_1.png)


PEFT 的核心思想：**冻结大部分参数，只训练少量新参数**。

| 方法 | 原理 | 可训练参数占比 |
|------|------|--------------|
| Prefix Tuning | 在输入前加可学习的"前缀"向量 | ~0.1% |
| Prompt Tuning | 只学 soft prompt embedding | ~0.01% |
| Adapter | 在 Transformer 层间插入小模块 | ~1-5% |
| **LoRA** | 低秩分解权重矩阵 | ~0.1-1% |

**LoRA 是当前最主流的 PEFT 方法**。

## LoRA 原理详解

![LoRA 原理示意图](/images/ch36_11_1_2.png)


### 核心思想

预训练模型的权重更新矩阵 $\Delta W$ 通常是**低秩**的——它不需要满秩的 $d \times d$ 矩阵来表示。

LoRA 把 $\Delta W$ 分解为两个小矩阵：

$$W = W_0 + \Delta W = W_0 + BA$$

其中：
- $W_0 \in \mathbb{R}^{d \times d}$：冻结的预训练权重
- $B \in \mathbb{R}^{d \times r}$：可训练的低秩矩阵
- $A \in \mathbb{R}^{r \times d}$：可训练的低秩矩阵
- $r \ll d$：秩（通常 8、16、32、64）

### 参数量对比

全量微调：$d \times d = 4096 \times 4096 = 16.7M$ 参数

LoRA（r=8）：$d \times r + r \times d = 4096 \times 8 + 8 \times 4096 = 65K$ 参数

**参数量减少 256 倍！**

### 前向传播

```python
# 原始前向传播
# h = W_0 @ x

# LoRA 前向传播
# h = W_0 @ x + (B @ A) @ x
# 其中 W_0 冻结，只有 B 和 A 参与梯度更新
```

### 缩放因子

$$h = W_0 x + \frac{\alpha}{r} BAx$$

$\alpha$ 是缩放因子，控制 LoRA 的影响强度。通常 $\alpha = 2r$。

## LoRA PyTorch 实现

```python
import torch
import torch.nn as nn


class LoRALinear(nn.Module):
    """LoRA 包装的线性层"""

    def __init__(self, original_layer, r=8, alpha=16):
        super().__init__()
        self.original = original_layer
        self.original.weight.requires_grad = False
        if self.original.bias is not None:
            self.original.bias.requires_grad = False

        in_features = original_layer.in_features
        out_features = original_layer.out_features

        # LoRA 矩阵
        self.lora_A = nn.Parameter(torch.randn(r, in_features) * 0.01)
        self.lora_B = nn.Parameter(torch.zeros(out_features, r))
        self.scaling = alpha / r

    def forward(self, x):
        # 原始输出 + LoRA 输出
        original_out = self.original(x)
        lora_out = (x @ self.lora_A.T @ self.lora_B.T) * self.scaling
        return original_out + lora_out


# 使用
layer = nn.Linear(4096, 4096)
lora_layer = LoRALinear(layer, r=8, alpha=16)

# 只有 lora_A 和 lora_B 有梯度
trainable = sum(p.numel() for p in lora_layer.parameters() if p.requires_grad)
total = sum(p.numel() for p in lora_layer.parameters())
print(f"可训练参数: {trainable:,} / {total:,} = {trainable/total:.2%}")
```

### 应用到注意力层

LoRA 通常应用到 Q、K、V、O 投影矩阵：

```python
def apply_lora_to_model(model, r=8, alpha=16, target_modules=None):
    """给模型的注意力层加 LoRA"""
    if target_modules is None:
        target_modules = ["q_proj", "k_proj", "v_proj", "o_proj"]

    for name, module in model.named_modules():
        for target in target_modules:
            if name.endswith(target):
                # 替换为 LoRA 版本
                parent = dict(model.named_modules())[name.rsplit(".", 1)[0]]
                lora = LoRALinear(module, r=r, alpha=alpha)
                setattr(parent, target.split(".")[-1], lora)
    return model
```

## QLoRA（4-bit 量化 + LoRA）

QLoRA = 4-bit NF4 量化基础模型 + LoRA 微调。显存需求从 100GB+ 降到 **~12GB**（7B 模型）。

### 原理

1. 把预训练权重从 FP16 量化到 NF4（4-bit NormalFloat）
2. LoRA 的 A、B 矩阵保持 FP16/BF16
3. 前向传播时反量化 → 计算 → 量化

```python
from transformers import BitsAndBytesConfig

# 4-bit 量化配置
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_compute_dtype=torch.bfloat16,
    bnb_4bit_use_double_quant=True,  # 双重量化，进一步节省
)

# 加载量化模型
model = AutoModelForCausalLM.from_pretrained(
    "Qwen/Qwen2.5-7B",
    quantization_config=bnb_config,
    device_map="auto",
)
```

## 用 PEFT 库实战

HuggingFace PEFT 库提供了标准化的 LoRA 接口：

```python
from peft import LoraConfig, get_peft_model, TaskType

# LoRA 配置
lora_config = LoraConfig(
    task_type=TaskType.CAUSAL_LM,
    r=8,                     # 秩
    lora_alpha=32,           # 缩放因子
    lora_dropout=0.05,       # Dropout
    target_modules=[         # 应用 LoRA 的层
        "q_proj", "k_proj", "v_proj", "o_proj",
        "gate_proj", "up_proj", "down_proj",
    ],
    bias="none",
)

# 应用 LoRA
model = get_peft_model(model, lora_config)
model.print_trainable_parameters()
# trainable params: 4,194,304 || all params: 1,547,845,632 || trainable%: 0.27%

# 训练...
# 保存（只保存 LoRA 权重，~16MB 而非 14GB）
model.save_pretrained("output/lora_weights")

# 加载
from peft import PeftModel
base_model = AutoModelForCausalLM.from_pretrained("Qwen/Qwen2.5-1.5B")
model = PeftModel.from_pretrained(base_model, "output/lora_weights")
```

## 模型量化

### 量化原理

把高精度浮点数（FP32/FP16）映射为低精度整数（INT8/INT4）：

$$q = \text{round}\left(\frac{x}{\text{scale}}\right) + \text{zero\_point}$$

### 主流量化方法

| 方法 | 位数 | 特点 |
|------|------|------|
| GPTQ | 4-bit | 需要校准数据集，推理快 |
| AWQ | 4-bit | 保护关键权重，质量更好 |
| GGUF | 2-8-bit | llama.cpp 格式，CPU 友好 |
| bitsandbytes | 4/8-bit | 训练时动态量化，QLoRA 标配 |

### GGUF 格式（llama.cpp）

```bash
# 转换为 GGUF
python convert_hf_to_gguf.py model_name --outfile model.gguf

# 量化为 Q4_K_M
./llama-quantize model.gguf model-q4km.gguf Q4_K_M

# 推理
./llama-cli -m model-q4km.gguf -p "Hello" -n 100
```

### 量化对性能的影响

| 精度 | 模型大小（7B） | 推理速度 | 质量损失 |
|------|--------------|---------|---------|
| FP16 | 14GB | 基准 | 无 |
| INT8 | 7GB | ~1x | 极小 |
| INT4 | 3.5GB | ~1.5x | 轻微 |
| INT2 | 1.75GB | ~2x | 明显 |

## 常见误区

1. **秩越大越好**：r=8 通常够用，r=64 可能过拟合
2. **只在 Q/V 上加 LoRA**：Q/K/V/O + MLP 都加效果更好
3. **量化无损**：INT4 有质量损失，关键任务用 INT8 或 FP16
4. **LoRA 能替代全量微调**：数据量大时全量微调仍更好
5. **忽略学习率**：LoRA 学习率通常比全量微调大（1e-4 ~ 3e-4）

## 小结

| 概念 | 说明 |
|------|------|
| PEFT | 只训练少量参数，冻结大部分预训练权重 |
| LoRA | 低秩分解 $\Delta W = BA$，参数量减少 ~256 倍 |
| QLoRA | 4-bit 量化 + LoRA，7B 模型只需 ~12GB 显存 |
| 秩 r | 控制 LoRA 容量，通常 8-64 |
| 量化 | FP16→INT8/INT4，缩小模型、加速推理 |
