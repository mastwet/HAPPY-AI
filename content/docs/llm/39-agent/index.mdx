# 39 Agent

如果说 RAG 让 LLM 能"查资料"，那 Agent 就让 LLM 能**"使用工具"**——搜索网页、执行代码、调用 API、操作数据库。Agent 是 LLM 从"聊天机器人"进化为"智能助手"的关键。

## 什么是 Agent

![Agent 工作原理](/images/ch39_7-3-Agent工作原理.png)


Agent = LLM + 工具 + 记忆 + 规划

| 组件 | 作用 | 例子 |
|------|------|------|
| LLM（大脑） | 理解指令、推理、决策 | GPT-4、Qwen |
| 工具（手脚） | 与外部世界交互 | 搜索、代码执行、API |
| 记忆（记忆） | 存储对话历史和长期知识 | 对话 buffer、向量存储 |
| 规划（思维） | 分解任务、制定计划 | ReAct、Plan-and-Execute |

## Agent 的发展历程

```text
2022 ChatGPT Plugins    → 首次让 LLM 使用工具
2023 AutoGPT / BabyAGI  → 自主 Agent 概念爆发
2023 LangChain Agent    → 工程化 Agent 框架
2024 GPT-4 Turbo        → 原生 Function Calling
2024 Claude MCP         → 标准化工具协议
2025 OpenAI Codex       → 编程 Agent
```

## ReAct 框架

ReAct（Reasoning + Acting）是 Agent 的经典范式：**交替进行思考和行动**。

```text
Question: 北京今天天气怎么样？

Thought 1: 我需要查询北京今天的天气，应该用搜索工具。
Action 1: search("北京今天天气")
Observation 1: 北京今天晴，气温 15-25°C，东南风 3 级。

Thought 2: 我已经获取到天气信息，可以回答用户了。
Action 2: finish("北京今天天气晴朗，气温 15-25°C，东南风 3 级，适合外出。")
```

**关键**：每一步都是"想→做→观察"的循环，直到任务完成。

## Function Calling

Function Calling 是让 LLM 调用外部函数的标准方式。

### 定义工具

```python
tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "获取指定城市的天气信息",
            "parameters": {
                "type": "object",
                "properties": {
                    "city": {
                        "type": "string",
                        "description": "城市名称，如'北京'",
                    },
                    "date": {
                        "type": "string",
                        "description": "日期，格式 YYYY-MM-DD，默认今天",
                    },
                },
                "required": ["city"],
            },
        },
    },
]
```

### 调用流程

```python
from openai import OpenAI

client = OpenAI()

# 1. 用户提问
messages = [{"role": "user", "content": "北京今天天气怎么样？"}]

# 2. LLM 决定调用哪个工具
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=messages,
    tools=tools,
    tool_choice="auto",
)

# 3. 解析 LLM 的工具调用
tool_call = response.choices[0].message.tool_calls[0]
function_name = tool_call.function.name
arguments = json.loads(tool_call.function.arguments)
print(f"调用: {function_name}({arguments})")
# 调用: get_weather({"city": "北京"})

# 4. 执行工具，获取结果
weather_data = get_weather(**arguments)

# 5. 把结果返回给 LLM
messages.append(response.choices[0].message)
messages.append({
    "role": "tool",
    "tool_call_id": tool_call.id,
    "content": json.dumps(weather_data),
})

# 6. LLM 生成最终回答
final_response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=messages,
)
print(final_response.choices[0].message.content)
```

### 并行工具调用

LLM 可以在一次回复中调用多个工具：

```json
{
    "tool_calls": [
        {"function": {"name": "get_weather", "arguments": "{\"city\": \"北京\"}"}},
        {"function": {"name": "get_weather", "arguments": "{\"city\": \"上海\"}"}}
    ]
}
```

## 构建一个完整的 Agent

```python
import json
from openai import OpenAI

client = OpenAI()


def run_agent(user_message, tools, tool_functions, max_steps=10):
    """运行 Agent 循环"""
    messages = [{"role": "user", "content": user_message}]

    for step in range(max_steps):
        # LLM 思考
        response = client.chat.completions.create(
            model="gpt-4o-mini",
            messages=messages,
            tools=tools,
            tool_choice="auto",
        )

        msg = response.choices[0].message
        messages.append(msg)

        # 如果没有工具调用，任务完成
        if not msg.tool_calls:
            return msg.content

        # 执行所有工具调用
        for tool_call in msg.tool_calls:
            func_name = tool_call.function.name
            func_args = json.loads(tool_call.function.arguments)

            # 执行工具
            if func_name in tool_functions:
                result = tool_functions[func_name](**func_args)
            else:
                result = f"Error: unknown function {func_name}"

            # 把结果返回给 LLM
            messages.append({
                "role": "tool",
                "tool_call_id": tool_call.id,
                "content": str(result),
            })

    return "达到最大步骤数，任务未完成"


# 工具定义
def search(query):
    return f"搜索结果: {query} 的相关信息..."

def calculate(expression):
    return eval(expression)

tool_functions = {"search": search, "calculate": calculate}

# 运行
result = run_agent("计算 sin(30°) 的值", tools, tool_functions)
print(result)
```

## LangChain Agent

LangChain 提供了更高级的 Agent 抽象：

```python
from langchain_openai import ChatOpenAI
from langchain.agents import AgentExecutor, create_tool_calling_agent
from langchain_core.tools import tool
from langchain_core.prompts import ChatPromptTemplate

# 定义工具
@tool
def search(query: str) -> str:
    """搜索互联网获取最新信息"""
    return f"搜索结果: {query}"

@tool
def calculator(expression: str) -> str:
    """计算数学表达式"""
    return str(eval(expression))

# 创建 Agent
llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个有用的助手，可以使用工具来回答问题。"),
    ("human", "{input}"),
    ("placeholder", "{agent_scratchpad}"),
])

agent = create_tool_calling_agent(llm, [search, calculator], prompt)
executor = AgentExecutor(agent=agent, tools=[search, calculator], verbose=True)

# 运行
result = executor.invoke({"input": "今天北京天气怎么样？"})
print(result["output"])
```

## MCP 协议

MCP（Model Context Protocol）是 Anthropic 提出的标准化工具协议：

```text
传统方式：每个 LLM 平台需要适配每个工具
MCP 方式：工具实现一次 MCP Server，所有 LLM 平台都能用
```

MCP 的核心思想：**工具是标准化的"服务器"，Agent 是"客户端"**。

```text
Agent (Client) ←→ MCP Protocol ←→ Tool Server (搜索/数据库/API)
```

MCP 定义了三种能力：
- **Tools**：可调用的函数
- **Resources**：可读取的数据
- **Prompts**：预定义的提示模板

## Agent 的局限与挑战

| 挑战 | 说明 |
|------|------|
| 幻觉 | Agent 可能编造工具调用参数 |
| 循环 | 反复调用同一工具，陷入死循环 |
| 规划失败 | 复杂任务分解不正确 |
| 安全 | 工具可能执行危险操作（删文件、发邮件） |
| 成本 | 多步推理消耗大量 Token |
| 延迟 | 每步都需要 LLM 推理，响应慢 |

**安全最佳实践**：
- 工具调用前人工确认（Human-in-the-Loop）
- 限制工具权限（只读 vs 读写）
- 设置最大步骤数
- 日志审计

## 常见误区

1. **Agent 万能论**：简单任务不需要 Agent，直接 Prompt 就行
2. **忽视错误处理**：工具调用可能失败，必须有 fallback
3. **工具太多**：工具太多反而让 LLM 困惑，控制在 10-20 个
4. **不做评估**：Agent 行为不确定，必须有评估指标
5. **忽视成本**：Agent 的 Token 消耗是普通对话的 5-10 倍

## 小结

| 概念 | 说明 |
|------|------|
| Agent | LLM + 工具 + 记忆 + 规划 |
| ReAct | 交替思考（Reasoning）和行动（Acting） |
| Function Calling | LLM 调用外部函数的标准方式 |
| MCP | Anthropic 提出的标准化工具协议 |
| Human-in-the-Loop | 关键操作人工确认 |
